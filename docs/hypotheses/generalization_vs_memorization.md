# Hypothesis: Multi-Rule Collapse Prevents Generalization

**Date:** 2026-02-13
**Status:** Active — empirically motivated, not yet formally tested
**Authors:** Project team

## The Hypothesis

When a neural network is trained on data generated by multiple simple rules, it may interpret the mixture as a single complex (noisy) rule rather than recognizing the existence of multiple distinct simple rules. This causes the network to memorize training examples rather than generalize, because the "one complex rule" model cannot extrapolate to unseen inputs — only a model that has identified the individual simple rules can do that.

**Corollary on grokking:** The delayed generalization phenomenon known as "grokking" may correspond to the network eventually discovering that the data decomposes into multiple simple rules rather than one complex one. This would explain why grokking is abrupt (a discrete structural insight) and why it requires extended training far beyond the memorization point.

## How We Arrived at This Hypothesis

### Step 1: Stage 1 results — single rules generalize immediately

In Stage 1 of the ePC-JEPA evaluation (experiments/energy_reasoning/), the model was trained on single-rule tasks: given input-output pairs following one rule (e.g., "double"), predict the output for a new input.

**Key observation:** Training and test accuracy rose in near-perfect lockstep. The model generalized almost as fast as it memorized. By epoch 10, both train and test accuracy exceeded 96%. There was essentially no generalization gap.

This held across all Stage 1 sub-tests (1a, 1b, 1c) with different rules and configurations. Single rules are trivially learnable.

### Step 2: Stage 2 results — multiple rules cause catastrophic overfitting

Stage 2 introduced pattern induction: the model sees in-context examples from one of 5 rules (double, shift3, square, complement, shift7) and must infer which rule applies, then predict the output for a query input.

**Key observation:** Training accuracy reached 99%+, but test accuracy plateaued at approximately 25%. The model memorized the training set completely but could not generalize to unseen examples at all.

The 4 ablation variants all showed similar test accuracy:
- no_z (no rule hint): ~25-28%
- random_z: ~25-28%
- oracle_z (correct rule vector): ~20-27%
- langevin (energy-minimized z): ~20-25%

The oracle z result is especially telling: even when explicitly given a vector identifying the rule, the model performed no better than without it. The predictor learned to completely ignore the rule indicator.

### Step 3: The 25% observation triggers the insight

25% accuracy with 5 rules is suspiciously close to 1/5 (20%) or 1/4 (25%). This suggests the model learned approximately one rule well and applies it uniformly to all test examples, regardless of which rule actually generated them.

The reasoning chain:
1. Single rules → instant generalization (Stage 1)
2. Five rules → 99% train, ~25% test (Stage 2)
3. 25% ≈ performance if you perfectly learned 1 out of 5 rules
4. Therefore: the model probably learned one rule, not five
5. **Why?** Because it never learned that there ARE five rules — it interpreted the mixture as one complex noisy function
6. A complex noisy function can only be memorized, not generalized
7. This explains why oracle z is ignored: if you don't know multiple rules exist, a rule indicator is meaningless

### Step 4: Connection to grokking

This reframes grokking: the long delay between memorization and generalization isn't the model "slowly getting better" — it's the model maintaining a wrong structural assumption (one complex rule) until something triggers the discrete insight that the data contains multiple simple rules. The abruptness of grokking is explained by this being a qualitative structural transition, not a gradual quantitative improvement.

## Empirical Evidence

### Direct evidence (our experiments)

| Experiment | Rules | Train Acc | Test Acc | Generalization Gap | Interpretation |
|-----------|-------|-----------|----------|-------------------|----------------|
| Stage 1a | 1 | ~97% | ~97% | ~0% | Single rule: instant generalization |
| Stage 1b | 1 | ~97% | ~97% | ~0% | Confirms across rules |
| Stage 1c | 1 | ~97% | ~97% | ~0% | Confirms across rules |
| Stage 2 | 5 | 99%+ | ~25% | ~74% | Multi-rule: catastrophic overfitting |

### Ablation evidence

| Variant | Test Acc | Implication |
|---------|----------|-------------|
| no_z | ~25-28% | Baseline: no rule information |
| oracle_z | ~20-27% | Model ignores explicit rule hint |
| langevin | ~20-25% | Energy minimization over z doesn't help |
| random_z | ~25-28% | Random z ≈ no z (confirms z is ignored) |

The oracle z result is the strongest evidence. If the model understood that multiple rules exist but merely struggled to identify which one, oracle z would dramatically improve performance. The fact that it doesn't means the model hasn't learned the multi-rule structure at all.

### Langevin gap

The Langevin gap (accuracy_langevin - accuracy_no_z) was consistently negative (-5% to -7%). This means energy minimization over z actively hurts, likely because the energy landscape over z has no meaningful structure — there's no "correct z" to find when the model doesn't condition on z.

## Supporting Research

### The Quantization Model of Neural Scaling (Michaud et al., NeurIPS 2023)

**Link:** https://arxiv.org/abs/2303.13506

Networks learn discrete, indivisible skills ("quanta") — either fully learned or not learned at all. On multi-task datasets, skills are learned one at a time, roughly in order of frequency, each as a sharp discrete transition. Our observation that the model learns approximately one rule maps directly onto this: we're seeing the first quantum being learned. The model may eventually learn additional quanta with more training, data, or capacity.

### Grokked Transformers are Implicit Reasoners (Wang et al., NeurIPS 2024)

**Link:** https://arxiv.org/abs/2405.15071

Transformers learn implicit reasoning only through grokking. The model transitions from a memorizing circuit to a generalizing circuit. Critically, the ratio of inferred facts to atomic facts controls grokking speed. In our setup, all examples are equally complex — there's no "easy" subset that teaches rule structure, which may explain why grokking hasn't occurred even after 50 epochs.

### Towards Understanding Grokking (Liu et al., NeurIPS 2022)

**Link:** https://arxiv.org/abs/2205.10343

Provides a theoretical framework for grokking as representation learning. The model first memorizes using a high-complexity representation, then transitions to a low-complexity generalizing representation. Our hypothesis adds specificity: in multi-rule settings, the "high-complexity representation" is the model's attempt to fit one complex rule, and the "low-complexity representation" is the decomposition into multiple simple rules.

### Grokking: Generalization Beyond Overfitting (Power et al., 2022)

**Link:** https://arxiv.org/abs/2201.02177

The original grokking paper. Demonstrated that small algorithmic datasets (modular arithmetic) exhibit delayed generalization after complete memorization. Notably, their tasks (modular addition, etc.) can be decomposed into simpler sub-operations — consistent with our hypothesis that grokking involves discovering decomposition structure.

### Deep Grokking (Fan et al., 2024)

**Link:** https://arxiv.org/abs/2405.19454

Observed multi-stage grokking in deep networks: test accuracy improves through multiple sharp transitions rather than one. This is consistent with learning successive quanta — each stage could correspond to the network discovering one more rule in the mixture.

### The Complexity Dynamics of Grokking (deMoss et al., 2024)

**Link:** https://www.sciencedirect.com/science/article/pii/S0167278925003367

Measured network complexity through training and found a characteristic rise-and-fall: complexity increases during memorization and drops sharply during generalization. The generalizing solution is simpler. In our framing: "5 simple rules" has lower complexity than "1 complex rule that fits all training data," so the transition to generalization is literally a simplification.

## Proposed Next Steps to Test the Hypothesis

1. **Curriculum learning (one rule at a time):** If the bottleneck is discovering multi-rule structure, training on rules sequentially should help. Train rule 1 to generalization, then add rule 2, etc. If this works, it confirms the model can learn each rule but fails at simultaneous discovery.

2. **Explicit rule token (not latent z):** Instead of a latent z that the model must learn to use, prepend an explicit rule-ID token to each sequence. This removes the "discover that multiple rules exist" problem entirely and tests whether the model can apply rules conditionally when told which one is active.

3. **Extended training (grokking test):** Train for 10-100x longer than the memorization point (500-5000 epochs instead of 50). If grokking occurs, it supports the hypothesis that the model CAN discover multi-rule structure given enough time. If it doesn't, capacity or architecture may be the bottleneck.

4. **Scaling rules gradually:** Test with 2 rules, 3 rules, 4 rules, 5 rules. If test accuracy degrades proportionally (50% → 33% → 25% → 20%), it confirms the "learns exactly one rule" pattern and rules out alternative explanations.

5. **Data ratio manipulation:** Following Wang et al., vary the ratio of "easy" examples (many in-context pairs, making the rule obvious) to "hard" examples (few pairs). If more easy examples accelerate grokking, it confirms that rule identifiability is the bottleneck.
