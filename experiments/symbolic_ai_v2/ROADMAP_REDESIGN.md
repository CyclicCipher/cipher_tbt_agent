## Part I — Category Theory Notes

### I.1 What Graph-SEQUITUR is, categorically

SEQUITUR computes a quotient of the **free monoid** Σ\* by identifying repeated pairs. The free monoid is the initial object in the category of monoids: every monoid homomorphism Σ\* → M is uniquely determined by a function Σ → M. SEQUITUR discovers a quotient monoid M = Σ\*/~ by finding the smallest congruence supported by the data.

This is the right starting idea, but **monoids are the wrong algebraic structure**. They have one operation (concatenation), one arity (binary), and no type system. Every failure in the current system — bigram-only compositions, no multi-scale structure, no cross-domain transfer, arithmetic blind to token semantics — traces to this choice.

### I.2 The monad determines the algebra

The structure to learn is determined by asking: **what is the monad whose Eilenberg-Moore algebras are the things we want to learn?**

| Domain | Monad | EM algebra |
|---|---|---|
| Flat sequences | List monad `T(A) = A*` | Monoids (= what SEQUITUR learns) |
| Hierarchical sequences | Free tree monad `T_Σ` | Σ-algebras (= what grammars are) |
| Probabilistic sequences | Giry monad `G` | Convex spaces |
| Probabilistic hierarchical | `G ∘ T_Σ` | Probabilistic tree grammars (PCFGs, HMMs) |

An element of `T_Σ(A)` is a tree whose leaves are drawn from A and whose internal nodes are drawn from Σ. A production rule in a CFG is literally a morphism in the EM category of `T_Σ`. The **free T_Σ-algebra** on a set of atoms is the set of all derivation trees; learning grammar = finding the minimal quotient algebra consistent with observations.

**The EM algorithm** (expectation-maximization for PCFGs) is the algorithm for finding EM algebras of the Giry monad. The name is not a coincidence.

### I.3 The Eilenberg-Brzozowski duality: learning IS categorical

The **Myhill-Nerode theorem** stated categorically: two strings u, v are equivalent (u ~_L v) iff they have identical distributional contexts — ∀w: L(u·w) = L(v·w). The minimal automaton for L is the quotient Σ\*/~_L, which is the **image** of the language morphism `L: Σ* → 2` in the category of monoids.

For probabilistic data, replace `2` (the Boolean semiring) with `(ℝ≥0, +, ×)`. Two strings are equivalent when they are **spectrally indistinguishable**: same left singular vector in the Hankel matrix. The Hankel matrix `H[u, v] = P(uv)` is the **Hom-set** of the learned category. Its SVD is the morphism factorization in the EM category:

```
H = U · S · V^T
     ↑       ↑
  forward  backward
  context  context
  states   states
```

The singular values S are the **canonical state space** — derived from the universal property of the factorization, not chosen by a heuristic. This is spectral learning of weighted finite automata (Hsu-Kakade-Zhang 2009), and it is the Myhill-Nerode theorem computed in `Mat(ℝ≥0)` rather than `Set`.

### I.4 Multi-scale: replace the Hankel matrix with a Hankel tensor family

For structure at all scales simultaneously, build the tensor family:

```
H_1[u, v]        = P(u·v)               — bigram
H_k[u_1...u_k, v] = P(u_1·...·u_k · v)  — k-gram, for each k
```

These share a **common latent state space S** if the process is generated by a stochastic tree grammar. The condition:

```
H_k(u_1...u_k, v) ≈ Σ_s [U_k(u_1...u_k, s)] · [V(s, v)]
```

where `U_k` factors as a tree contraction — the composition of states at level k factors through states at level k-1. This is the tensor-train/MPS decomposition. It is not arbitrary: it is the condition for the process to be generated by a **regular tree automaton**, which is a coalgebra for the polynomial functor `F(Q) = Σ_{σ∈Σ} Q^{arity(σ)}`.

The **learning algorithm** is joint Tucker decomposition of the Hankel tensor family under the constraint that the core tensors form a consistent tree algebra. The shared state space S across all scales IS the object of the learned category. The Tucker factors ARE the morphisms.

For self-similar/fractal structure: use the **terminal coalgebra** of F rather than the initial algebra. The terminal coalgebra is the set of possibly-infinite trees, capturing infinite-depth recursion naturally.

### I.5 Cross-domain transfer: natural transformations

Two domains D₁, D₂ have categories C₁, C₂ with state spaces S₁, S₂. A cross-domain transfer is a **natural transformation** η: F₁ ⇒ F₂ between the learned functors. Concretely: a linear map η: S₁ → S₂ that commutes with all transition operators (an **intertwiner** between two representations).

For NL word problems: S_NL ⊗ S_Math has a subspace fixed by the natural transformation that identifies "the token '5' in Latin text" with "the integer 5 in arithmetic notation." This subspace is found by joint SVD of the mixed-domain Hankel tensor.

### I.6 Generalization: Kan extensions

The template lookup failure (7.5% Latin accuracy) is precisely the failure to compute a **left Kan extension**. The training lookup is a functor:

```
F: C_train → Answers,   (N1, N2) ↦ N1+N2
```

The left Kan extension `Lan_i(F): C_all → Answers` along the inclusion `i: C_train ↪ C_all` is:

```
(Lan_i F)(n1, n2) = colim_{(m1,m2) →[f] (n1,n2)} F(m1, m2)
```

This colimit IS the addition function — computed by chaining successor morphisms from training examples. But to compute it, C_all must have morphisms (integers have successor: n → n+1). The current template system has no morphism structure on its state space — it is a set of opaque ID pairs. That is why fold_detect cannot generalize: it treats atom IDs as objects with no morphisms between them.

The fix is structural: the state space S must be an object in a category with the right morphisms. For integer arithmetic, the right category is the natural numbers object. For language, it is the type lattice from FCA. The Kan extension then computes automatically from the morphism structure.

### I.7 Zero free parameters

The architecture has **zero free parameters** when the semiring is chosen:

| Choice | Options |
|---|---|
| Which monad? | List (flat), Tree (hierarchical), both via composition |
| Which semiring? | `ℝ≥0` (probability), `ℤ` (exact count), `𝔹` (membership) |
| What is Σ? | The observed vocabulary |

State space size, rule structure, and transfer maps all fall out of universal properties. Nothing is thresholded, nothing is hardcoded to a domain.

### I.8 Why this is "directly from" rather than "justified by"

- The Hankel matrix **is** the Hom-set of the learned category
- The SVD **is** the morphism factorization in `Mat(ℝ≥0)`
- The Tucker decomposition **is** the EM-algebra structure for the tree monad
- The Kan extension **is** the generalization map — uniquely defined by its universal property
- Cross-domain transfer **is** a natural transformation

Connection to quantum physics: MERA (multi-scale entanglement renormalization ansatz) was independently discovered for the same reason — learning multi-scale structure without choosing scales a priori. It is a tree tensor network, which is exactly a tree automaton EM algebra under Tucker decomposition.

## Part II — Diagnosis of Current Failures

### II.1 The two failing tests

```
TestThreeStageTransfer::test_latin_accuracy_A2   7.5%  (target ≥70%)
TestThreeStageTransfer::test_mhg_accuracy_A2    17.5%  (target ≥70%)
```

These cannot be fixed by patching the current template/fold/adjunction layer.

### II.2 Root cause: template pollution

`_collect_observations` in `templates.py` iterates **every** edge in `mg._out` — every position in every training sequence — and groups by `(etype, ctx_len)`. For a 15-token Latin sequence, a 5-atom group gets:

- `[N1, et, N2, quot, ?]` → `SUM` (what we want)
- `[et, N2, quot, ?, SUM]` → `<eos>` (next position in the same sequence)
- `[marcus, habet, poma, ., julia]` → `dat` (narrative mid-sequence)
- `[dat, ei, ., N1, et]` → `N2` (another mid-sequence position)

Anti-unification of these 22 diverse 5-atom contexts collapses to `[?0 ?1 ?2 ?3 ?4]` — all variables. The lookup becomes a hash of 22 arbitrary 5-tuples mapping to arbitrary next tokens, not a meaningful formula template.

### II.3 Root cause: fold_detect is semantically blind

Even with a correctly built template, `fold_detect` cannot generalize to unseen (N1, N2) pairs. The fold observations are `([N1_id, N2_id], sum_id)` where IDs are arbitrary registration-order integers, not the semantic values `{1..6}`. The relationship `sum = N1 + N2` exists in atom *values* (`mg.symbols[id].value = '5'`), not atom *IDs*. `fold_detect` never inspects `.value`.

### II.4 Root cause: no cross-domain chaining

The system separately learns:
- `add 5 2 eq 7` (from math training, math etype)
- `... 5 et 2 quot ? 7` (from Latin training, Latin etype)

These are stored as separate facts in different template groups. There is no mechanism to recognize that when a Latin word problem contains tokens '5' and '2', the math addition fact for (5, 2) is applicable. `functor.py` exists but is not wired into word problem prediction.

## Part III — Redesign Plan

### III.1 Core architectural change

Replace the **MorphismGraph observation pipeline** (which builds a bigram composition graph via Graph-SEQUITUR) with a **Hankel estimator** (which builds the Hom-set matrix of the learned category directly from corpus statistics).

The new pipeline:

```
Corpus sequences
      ↓
HankelEstimator.observe(seq, topo)     — accumulate co-occurrence counts
      ↓
HankelEstimator.build_state_space(k)   — Tucker decomposition of H_1...H_k
      ↓
StateSpace S                           — objects of the learned category
      ↓
TransitionSystem T: Σ × S → S         — morphisms of the learned category
      ↓
SpectralPredictor.predict(ctx, topo)   — Kan extension lookup + intertwiner
```

The `MorphismGraph` is NOT deleted immediately — it remains as the composition tracking layer for perplexity evaluation (it is correct for that use case). The new `HankelEstimator` is a parallel learning pathway that replaces the reasoning layer.

### III.2 Phase A — Hankel estimator core

**File:** `core/hankel.py`
**Replaces:** The learning aspect of `core/morphism.py` (Graph-SEQUITUR pairs/rules)
**Depends on:** `core/topology.py`

```python
class HankelEstimator:
    # H[u_tuple, v_str] = co-occurrence count
    # u_tuple: left context of length k (atom string values)
    # v_str:   single right-context atom (string value)

    def observe(self, seq: list[str], topo: Topology) -> None:
        """Accumulate Hankel counts from one sequence.
        For each position i and context width k=1..K_MAX:
          left_context  = seq[i-k:i]      (the k atoms before position i)
          right_context = seq[i]           (the atom at position i)
          H_k[left_context][right_context] += 1
        """

    def matrix(self, k: int) -> tuple[list[tuple], list[str], np.ndarray]:
        """Return (row_keys, col_keys, H_k_matrix) for context width k.
        Row = unique left context k-tuples.
        Col = unique right context atoms.
        H[i, j] = count(row_keys[i] → col_keys[j]).
        """

    def build_state_space(
        self,
        k_max: int = 4,
        rank: int | None = None,
    ) -> 'StateSpace':
        """Compute joint Tucker decomposition of H_1...H_k_max.

        rank=None: use the elbow in the singular value spectrum (automatic).
        Returns a StateSpace with shared state vectors and transition tensors.
        """
```

**Tests** (`tests/hankel_test.py`, 8 tests):
- `observe` on `['a','b','c','a','b','c']` builds H_2 with `(a,b) → c` having count 2
- `matrix(1)` returns a matrix whose rows sum to occurrence counts of each atom
- `build_state_space(rank=2)` on a bigram corpus with 2 underlying states recovers 2 states
- `build_state_space` on a random matrix produces `rank ≤ min(rows, cols)` states
- Perplexity of held-out data is lower with k=2 than k=1 on a structured corpus
- Two identical corpora produce identical state spaces (determinism)
- `observe` is O(K_MAX × len(seq)) per call
- `build_state_space` on 10K-token corpus completes in < 5 seconds

### III.3 Phase B — State space and spectral predictor

**File:** `core/state_space.py`
**Depends on:** Phase A

```python
@dataclass
class StateSpace:
    rank:       int                       # dimension of S
    U:          np.ndarray                # shape (n_left_contexts, rank)
    V:          np.ndarray                # shape (n_right_contexts, rank)
    S:          np.ndarray                # shape (rank,) — singular values
    row_index:  dict[tuple[str,...], int] # left context → row index
    col_index:  dict[str, int]            # right context atom → col index
    k:          int                       # context width this space was built from

    def encode(self, left_context: tuple[str,...]) -> np.ndarray:
        """Map a left context to a point in state space (forward encoding)."""

    def predict_dist(self, left_context: tuple[str,...]) -> dict[str, float]:
        """P(next | left_context) via U @ diag(S) @ V^T reconstruction."""

    def predict_unseen(
        self,
        left_context: tuple[str,...],
        fallback_atoms: list[str],
    ) -> dict[str, float]:
        """Kan extension: for contexts not in row_index, find the nearest
        neighbour in state space (minimum distance in U-embedding) and use
        its distribution.  'Nearest' = argmin ||encode(ctx) - U[i]||.
        This IS the left Kan extension colimit for metric spaces.
        """
```

**Tests** (`tests/state_space_test.py`, 6 tests):
- `predict_dist` on a seen context returns correct distribution
- `predict_unseen` on an unseen context returns a non-uniform distribution
- `predict_unseen` on a context adjacent (by one successor step) to a seen context returns a distribution closer to the seen context's distribution than the marginal
- State space dimension equals `rank` parameter
- `encode` is linear in the left context representation
- Round-trip: `encode(ctx) @ V^T ≈ H[ctx, :]` (reconstruction quality)

### III.4 Phase C — Cross-domain intertwiner

**File:** `reasoning/intertwiner.py`
**Depends on:** Phase B
**Replaces:** The cross-domain transfer metrics in `reasoning/functor.py`

An intertwiner between two domains is a linear map η: S₁ → S₂ such that:
```
η · T₁(σ) ≈ T₂(σ) · η    for all σ ∈ Σ₁ ∩ Σ₂
```
(It commutes with the shared-alphabet transitions.)

For NL-to-math transfer: Σ₁ ∩ Σ₂ = the digit tokens `{'0'..'9'}`. η maps the digit subspace of S_NL to the digit subspace of S_Math.

```python
def find_intertwiner(
    ss1: StateSpace,
    ss2: StateSpace,
    shared_atoms: list[str],
) -> np.ndarray:
    """Find the least-squares intertwiner matrix η: (rank1,) → (rank2,).

    Minimizes Σ_{σ ∈ shared_atoms} ||η · T1(σ) - T2(σ) · η||_F
    subject to ||η||_F = 1.

    This is a generalised Sylvester equation.  Solved via vec(η) = null(A)
    where A = Σ_σ (T1(σ)^T ⊗ I - I ⊗ T2(σ)).
    """

def transfer_predict(
    ctx:        tuple[str,...],
    ss_source:  StateSpace,
    ss_target:  StateSpace,
    eta:        np.ndarray,
) -> dict[str, float]:
    """Predict in target domain from a source-domain context.

    1. Encode ctx in source state space: v = ss_source.encode(ctx)
    2. Map to target state space: w = eta @ v
    3. Predict in target: w @ diag(S_target) @ V_target^T
    """
```

**Tests** (`tests/intertwiner_test.py`, 6 tests):
- Two identical domains produce identity intertwiner (up to rotation)
- Two disjoint domains produce near-zero intertwiner (no shared structure)
- `transfer_predict` on digit tokens from Latin space matches Math space predictions
- `find_intertwiner` is invariant to the orthogonal freedom in the SVD
- An intertwiner between S_Math and S_Latin, composed with the reverse intertwiner, is approximately identity on the shared subspace
- Runtime: `find_intertwiner` with rank ≤ 50 completes in < 1 second

### ~~III.5 Phase D — Integer-aware Kan extension for arithmetic~~ ❌ ILLEGAL — REMOVED

**Reason for removal:** This phase hard-wired the fact that `add 3 4 = 7` by embedding the Python
integer `+` operator into the prediction chain.  The human brain does not have an integer addition
subroutine — it discovers addition from examples.  Any approach that hard-codes domain knowledge
into the prediction loop is a shortcut, not learning.  Phase D is struck from the roadmap.

The arithmetic structure (successor, addition, multiplication, …) must emerge from the spectral
predictor's training data alone, the same way it emerges for every other domain.

### III.6 Phase E — Unified predictor wiring ✅ COMPLETE

**File:** `core/spectral_predict.py`
**Depends on:** Phases B, C, H
**Replaces:** The reasoning back-off chain in `core/predict.py` steps 0a–0f

The 5-level prediction chain (interleaved at each k):

```
For each k from k_max down to 1:
  1. Spectral exact lookup (Phase B):
     ss.predict_dist(atom_buf[-k:]) if the k-gram context is in raw_dist.
     Zero noise (raw MLE, not SVD reconstruction). Returns on hit.

  2. FrameSolver adjunction lookup (Phase H):
     frame_solver.predict(atom_buf[-k:]) — operator frame adjunction.
     Fires when ctx ends in 'eq' and an adjunction covers the query.
     Interleaved with exact lookup at same k to prevent short-context
     shadow (e.g. ('32','eq')→pred answer shadowing succ(32) query).

(After all k levels exhausted:)
  3. Spectral Kan extension (Phase B):
     ss.predict_unseen(atom_buf[-k:]) — nearest-neighbour in state space.

  4. Cross-domain transfer (Phase C):
     transfer_predict(atom_buf[-k:], ss_source, ss_target, eta)
     if a valid intertwiner is available for this context.

  5. Marginal:
     Uniform over all observed targets for this edge type.
```

### III.9 Phase H — FrameSolver: adjunction-based prediction ✅ COMPLETE

**File:** `reasoning/frame_solver.py`
**Depends on:** Phase B (StateSpace.raw_dist)
**Wired into:** Phase E (`SpectralPredictor`, Level 2, interleaved with exact lookup per k)

Discovers **operator frames** from `raw_dist` — deterministic mappings of the form
`op arg₁ … argₙ eq → answer` (contexts ending in `'eq'` with probability ≥ 0.95)
— and detects three categorical adjunction types between them:

```
Type U (arity 1):  op1(a)   = b  ↔  op2(b)   = a      e.g. succ ⊣ pred
Type A (arity 2):  op1(a,b) = c  ↔  op2(c,b) = a      e.g. add  ⊣ sub
Type B (arity 2):  op(a,b)  = c  ↔  op(a,c)  = b      e.g. sub self-adjoint
```

No hardcoded rules. No type checks. No arithmetic operators. A token named `'succ'`
is treated identically to one named `'red'` — only the co-occurrence geometry matters
(Yoneda lemma). Detection implements the categorical unit/counit coverage conditions.

**Math benchmark results** (verified 2026-03-11):

| Level       | Before FrameSolver | After FrameSolver | Novel accuracy | Mechanism |
|-------------|-------------------|-------------------|----------------|-----------|
| counting    | 100.0%            | 100.0%            | 100.0%         | exact     |
| successor   | 0.0%              | **90.0%**         | **90.0%**      | Type U (succ⊣pred) |
| subtraction | 0.0%              | **85.0%**         | **85.0%**      | Type B (sub self-adjoint) |
| addition    | 65.8%*            | 65.8%*            | 0.0%**         | no adjoint in corpus |
| multiplication | 65.8%*         | 65.8%*            | 0.0%**         | no adjoint in corpus |
| powers      | 7.7%              | 7.7%              | —              | requires log/root |
| integrals   | 100.0%            | 100.0%            | —              | memorised |

    Add⊣sub adjunction requires both operators in training. This is a corpus design

## Part IV — Implementation Notes

### IV.1 What NOT to do

- Do not hard-code domain knowledge into the prediction loop. Arithmetic structure must emerge
  from training data. Any shortcut that embeds Python arithmetic operators is illegal (see
  struck Phase D).
- Do not run training on this machine (Mistake #36).

### IV.2 Semiring choice for the implementation

Use `float64` (= ℝ≥0 under probability normalisation) for all Hankel matrices. The spectral decomposition uses NumPy's `np.linalg.svd`. No external dependencies beyond what is already installed.

### IV.3 Rank selection

The automatic rank selection rule (Phase A): use the **numerical rank** at relative tolerance 1e-3 × σ_max. This is the information-theoretic cutoff — singular values below this threshold encode noise, not structure. It requires zero free parameters.
