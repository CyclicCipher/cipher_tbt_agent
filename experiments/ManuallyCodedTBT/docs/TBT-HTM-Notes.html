TBT-HTM Notes

Thousand Brains Theory & Hierarchical Temporal Memory — sorted by topic

⚠ Terminology Disambiguation — Read First
Cortical column The large-scale biological unit: ~1mm diameter, 2.5mm deep, ~100,000 neurons, 6 layers, hundreds of minicolumns inside it. This is the unit that has a receptive field, votes with neighbors, and builds object models.
Minicolumn The canonical microcircuit inside a cortical column. There are hundreds of minicolumns per cortical column. In HTM spatial pooling, the output unit is a minicolumn — one bit in the spatial pooler's output SDR. A minicolumn has multiple cells; temporal memory operates at the cell level within minicolumns.
Cell The individual neuron inside a minicolumn. Temporal memory distinguishes cells within the same minicolumn to encode sequence context. Multiple cells per minicolumn = higher-order memory.
Layer One of the 6 horizontal strata inside a cortical column (L1–L6). Each layer contains thousands of pyramidal neurons. Layers are the computational stages of TBT.
01

Neocortex — Structure & General Properties

HTM + TBT
  • The neocortex evolved in mammals over the past 100–200 million years. It is the wrinkly outer layer of the brain — wrinkled to maximize surface area. Reptiles and birds lack a neocortex.
  • The neocortex is a homogeneous sheet: every part shares the same repeating circuit and solves every problem — regardless of modality or domain — in the same way.
  • There are approximately 150,000 cortical columns (~1mm diameter, ~2.5mm deep). Each cortical column contains ~100,000 neurons, ~500 million synapses (5,000 per neuron), dozens of cell types, and hundreds of minicolumns.
  • Total neocortex: ~15 billion neurons, ~75 trillion synapses.
  • The neocortex runs processing steps of approximately 5ms each.
  • Only 2% of neurons are active at any given time — sparse distributed activation.
  • The neocortex is divided into several dozen regions connected via nerve fibers through the white matter.
  • The reptilian brain handles metabolic function, emotions, drives, and reward. It sets goals; the neocortex plans and executes toward them.
  • The neocortex's primary function is to learn a model of the world, use it to plan goal-directed behavior, and execute those plans. Prediction is the primary training signal — you only learn when a prediction is wrong.
  • Two fundamental inputs to the neocortex: (1) a copy of outgoing motor commands, (2) a copy of incoming sensory data. Both are necessary so the neocortex can distinguish self-caused sensory change from externally-caused change.
02

Cortical Column — 6-Layer Architecture

TBT
  • Every cortical column shares the same 6-layer laminar structure, regardless of where it is in the neocortex or what modality it processes.
  • Every cortical column has a motor output, not just sensory columns.
  • Each cortical column is assigned to exactly one receptive field — a specific patch of sensory data it and only it attends to.

Layer Roles

  • Layer 1: Auxiliary biological role. Contains apical dendrite tufts from deep-layer neurons and long-range axons. Serves as a wiring substrate for top-down feedback signals. May not require separate computational modeling.
  • Layer 2 (upper L2/3): Receives input from L3 below it, and also receives input from L5a. This makes L2 an integration layer for both the processed sensory sequence (from L3) and the motor context (from L5a). Projects strongly to other cortical areas — L2 is the long-range corticocortical output layer.
  • Layer 3 (lower L2/3): Receives feedforward input primarily from L4. Projects to all layers of the home column (including feedback to L4), to L5a, and to neighboring columns' L3. The lateral projections for inter-column voting originate here. Also the site of sequence learning via temporal memory — stores sequences of feature-location pairs that have been bound upstream in L4.
  • Layer 4: Sensory input and feature-location binding layer. Thalamic relay cells deliver sensory data here. L4 also receives the location prediction from L6a — making L4 the site where "what" (sensory feature) and "where" (object-relative location) are first combined. L4 projects up to L3, carrying the bound feature-location representation. L4 also projects down to L6a (the connection between sensory input and the grid cell system).
  • Layer 5a: Intratelencephalic (IT) motor output. IT neurons have small cell bodies in upper L5. Receive from L3 and L6b. Project to other cortical columns, other cortical areas, and the striatum — this is the corticocortical signal that broadcasts the column's current sequence state to the rest of the neocortex. Also provides input back up to L2 (upper L2/3). L5a is NOT the displacement layer.
  • Layer 5b: Extratelencephalic (ET) motor output — Displacement Cell Modules. ET neurons have large "thick-tufted" cell bodies in lower L5. These are the displacement cells (Hawkins 2019). L5b has a modular structure that mirrors L6a: there is one displacement cell module for each grid cell module in L6a, with the same period structure. A single displacement cell module is ambiguous (it cannot uniquely represent a displacement, just as a single grid cell module cannot uniquely represent a position). Multiple displacement cell modules together uniquely represent a displacement — the same residue number system logic as L6a. Their axon splits: one branch goes to subcortical motor structures (the muscle command), the second branch goes to thalamic relay cells which become feedforward input to the next hierarchical cortical area. L5b makes bilateral connections with L6a grid cells — firing a displacement shifts each L6a module's phase by d mod λᵢ (path integration). Displacement cells serve two roles: (1) movement — updating the reference frame when the sensor moves, and (2) composition — representing the spatial relationship between two objects placed in each other's reference frames.
  • Layer 6a: Grid cells / reference frame. Tracks where the sensor is in the object's allocentric reference frame. Projects a location prediction to L4 (how the column anticipates what sensory input to expect given current position). Also projects feedback to the thalamic nucleus that provides sensory input to the column — modulatory only, does not drive thalamic firing, modifies how relay cells respond to subcortical input.
  • Layer 6b: Role not well understood. Projects to the same thalamic nucleus as L6a, and also forward to the next higher-order thalamic nucleus (routing to the next hierarchical level). Receives from L5a in the diagram. May have been a subplate remnant in evolutionary terms. Function in adult cortex actively under investigation.

The Binding Problem — Where Feature Meets Location

Layer 2/3 does not perform the initial feature-location binding. That binding happens at Layer 4, which receives both sensory input (from the thalamus/encoder) and the location prediction from L6a. L4's output to L3 already carries the combined feature-location representation. L3 then learns sequences of these bound pairs via temporal memory.

Dendritic Input Types (Per Pyramidal Neuron)

  • Proximal dendrites: Feedforward input from the layer below or from sensory input. Causes action potentials directly.
  • Basal/distal dendrites: Lateral context from the same layer. Modulatory — affects how the neuron responds to proximal input. Provides temporal context (which cells were active previously) and location context (from L6a).
  • Apical dendrites: Feedback from higher layers. Also modulatory. Carries top-down predictions.
A pyramidal neuron integrates three signal streams: feedforward (proximal), same-layer context (basal), and top-down feedback (apical). This is what allows a cell to be in a "predictive state" before it fires.

Inter-Column Connections

  • L3 → neighboring columns' L3: Lateral voting projections within the same cortical area. Columns whose receptive fields are spatially adjacent share their object representations here.
  • L2 → other cortical areas: Long-range corticocortical projections. L2 carries the integrated sensorimotor output of the column to distant regions.
  • L5a → other cortical columns and striatum: Motor context signal shared across columns and into the basal ganglia.
  • L5b → thalamus → next cortical area: The L5b motor output copy reaches the next hierarchical cortical area via a thalamic relay, forming the hierarchical processing chain.

Open Questions

  • What exactly is L6b doing? Its role in adult cortex is genuinely unclear. Does it receive from L5a as the diagram suggests, and if so what does it compute?
  • How does L6a's location prediction to L4 work mechanistically? Is it a gating signal, an additive modulatory input, or something else?
  • L5b is confirmed as the displacement cell layer (Hawkins 2019). The remaining question: how does L5b decide which displacement to output? Answer: via L5a, which is the conjunctive readout of (L3 context, L4 feature) → displacement. L5a drives L5b locally. See Phase 10 in Architecture Plan.
  • Some research indicates human minicolumns build group-theoretic representations. What does this mean computationally, and how should it inform the architecture?

Key Finding — Reset Burst Problem

A software reset (col.reset()) zeros all TM cell state, causing the column to burst at position 0 of every new context — even for well-trained positions. This has no biological analog. In the real neocortex, neurons don't spontaneously lose their dendritic synapse patterns between trials. The correct behavior is for the location signal alone (from L6a) to make the right cells predictive at any position, without sequential context being required.
  • Root cause: Location synapses in our TM are additive to cell synapses. After reset, prev_active is all zeros, so cell synapses contribute nothing and location synapses rarely reach threshold alone. Every first step bursts.
  • Correct biological behavior: The location signal arrives on basal/distal dendrites and independently depolarizes cells into a "ready to fire" state (dendritic spike). When proximal sensory input then arrives, pre-primed cells fire selectively without needing sequential context. The reference frame must be position-addressable, not walk-dependent.
  • Fix (Phase 10): Add a location_activation_threshold parameter to TemporalMemory. A segment becomes predictive if location synapses alone ≥ this threshold, regardless of cell synapse activity. This also resolves the multi-stride training competition problem from Phase 9a — if location alone primes cells, strides don't compete for burst-winner slots.

Key Finding — Phase 9b Results

  • Without a learned L5 displacement output, the column can only rote-memorize specific (a,b)→result pairs. Unique-result pairs (3+5=8, 2+7=9) memorize correctly (0.000 anomaly). Ambiguous-result pairs (1+9=10 and 5+5=10 share a result) fail completely.
  • The failure mode is architecturally revealing: without the reference frame being moved by L5b, both "1+9=10" and "5+5=10" sequences end at the same grid position (position 0, never moved), so the column has no basis to distinguish them. The reference frame IS what enables compositional addition.

Sparse Distributed Representations (SDRs)

HTM + TBT
  • The state of any region of the neocortex can be represented as a bit array — 1 if a neuron is active, 0 if not. Only ~2% are active: this is a Sparse Distributed Representation.
  • SDRs have two key parameters: length n (total bits) and population w (number of active bits). Capacity = n! / (w!(n−w)!). Example: n=256, w=4 → ~8.8 billion unique states.
  • SDRs are extremely noise-resistant: overlap score (AND) can match two SDRs even with multiple flipped bits. Threshold θ is a hyperparameter for minimum overlap to declare a match.
  • False positive rates at typical SDR sizes and noise tolerances are negligibly small — effectively zero.
  • Smallest HTM SDRs: length 2048, population 40. Largest used: 65536.
  • Union (OR): Combines a set of SDRs into one. Used for fast approximate membership testing. Saturates as more SDRs are added — use unions instead of O(n²) pairwise comparison.
  • Overlap (AND): Counts shared active bits between two SDRs. This is the semantic similarity metric.
  • SDRs can be concatenated — sub-encoders produce independent SDRs which are joined to form a combined representation (e.g. a datetime encoder concatenating day-of-week, time-of-day, and season sub-encoders).
  • Axon bundles can merge and split without data loss because of SDR noise resistance and semantic properties.

Encoding Rules

  • Semantically similar inputs must have high bit overlap. The same input must always produce the same output. Output length and sparsity must be consistent.
  • Scalar encoding: Encodes a continuous value as a consecutive bucket of on-bits within a range [min, max]. Adjacent scalar values share overlapping bits.
  • Periodic encoding: Like scalar, but wraps — max and min are semantically similar. Good for cyclic data (time of day, compass heading).
  • Random Distributed Scalar Encoding (RDSE): Scatters the active bits randomly but maintains overlap between adjacent values. Requires state, which is inconvenient for data-oriented design.
  • Multi-quantity encoders: Correlated quantities (e.g. systolic and diastolic blood pressure) should be encoded together or with explicit sub-encoders for their relationships, not independently.
04

Spatial Pooling

HTM

Column vs. Minicolumn Clarification

  • The spatial pooler operates at the minicolumn level. Its output is a boolean vector over minicolumns — which minicolumns won inhibition. It is entirely blind to individual cells within minicolumns.
  • Temporal memory then takes over and determines which cell within each active minicolumn fires based on sequence context. TM's output is a boolean vector over cells — a much longer vector of shape num_minicolumns × cells_per_minicolumn.
  • BAMI uses "column" and "minicolumn" interchangeably, which is a known source of confusion. The peer-reviewed paper (Cui et al. 2017) consistently uses "mini-column" for the SP output unit. Our code uses num_minicolumns throughout, which is correct.

Algorithm

  • Takes an input SDR of arbitrary size and translates it into a fixed-size output SDR of active minicolumns, maintaining fixed sparsity and preserving semantic overlap from the input.
  • Each minicolumn has a potential pool of input bits it might connect to, and a permanence value per potential connection. If permanence ≥ threshold, the synapse is considered connected.
  • Minicolumn activation: compute raw overlap score (connected synapses hitting active input bits), multiply by boost factor, then top-k inhibition. Only minicolumns whose raw overlap exceeds stimulus_threshold are eligible — this prevents noise columns from winning via boost alone.
  • Phase order: Boosting (multiply overlaps by boost factors) → Inhibition (top-k selection) → Hebbian learning (active columns only) → Duty cycle update → Boost factor update → Permanence boost (if needed).
  • Hebbian learning: For active minicolumns only — increase permanence for connections to active input bits, decrease for inactive. Non-winning columns are not modified at all.

Two Boosting Mechanisms

  • Boost factor (mechanism 1): Multiplies each minicolumn's overlap score before inhibition. Computed as exp(β × (mean_neighbor_duty_cycle − own_duty_cycle)). The reference is the mean active duty cycle of neighboring columns (global mean under global inhibition) — not a fixed target rate. This is an adaptive target that evolves with learning. Columns below the mean are boosted above 1; columns above are suppressed below 1.
  • Permanence boost (mechanism 2): If a minicolumn's overlap duty cycle (how often its raw overlap exceeds stimulus_threshold) falls below min_pct_overlap_duty_cycle × max(neighbors' overlap duty cycles), all its potential permanences are increased by 0.1 × permanence_threshold. This rescues columns that have lost all useful connections and can no longer even reach stimulus_threshold — they would never recover via Hebbian learning alone because they never win.
  • Without both mechanisms, ~30% of minicolumns end up never used. With both, nearly every minicolumn participates at its target activation rate. The peer-reviewed paper empirically confirms that learning without boosting performs no better than a random untrained SP.

Topology

  • Global inhibition: Every minicolumn competes with every other. Fastest and most common in practice.
  • Local inhibition with topology: Each minicolumn has a receptive field — a spatial window into the input space. Minicolumns only connect within their receptive field. Receptive fields overlap. Analogous to convolutions in CNNs. Important for vision and other sensory data; the learned representation can physically resemble the input space structure. Effects become more useful as input size increases.

Parameter Guidance — Source Variation

  • Parameter recommendations differ between Numenta sources. Both are authoritative; BAMI (v0.56, corrected Jan 2024) is more recent than the peer-reviewed paper (Cui et al. 2017). Neither set should be treated as fixed — empirical tuning is required for each domain.
  • num_minicolumns: BAMI minimum 2048. Target ~2% sparsity: active_per_step ≈ 0.02 × num_minicolumns.
  • permanence_threshold: BAMI recommends 0.2; paper uses 0.5. System is not sensitive to this parameter per BAMI.
  • permanence_inc / permanence_dec: BAMI: 0.03 / 0.015. Paper: 0.1 / 0.02. The ratio of dec to inc affects how aggressively the SP prunes connections — a higher dec/inc ratio produces sparser, more selective receptive fields.
  • boost_strength (β): BAMI does not specify a number; paper uses β=100. Our implementation defaults to 3.0, which is likely too mild — columns are barely differentiated. A value in the range 10–100 is more likely to produce meaningful homeostatic pressure. Treat as a tuning parameter.
  • Permanence initialization: Our implementation uses a normal distribution centered on the threshold (requires scipy). The paper uses uniform U(0,1). Both produce ~50% initial connectivity at threshold=0.5. The uniform approach is simpler and eliminates the scipy dependency.

Temporal Memory

HTM
  • Operates on the output of spatial pooling (the active minicolumns). Learns sequences of active minicolumn patterns and predicts the next pattern using temporal context.
  • Temporal memory distinguishes individual cells within minicolumns. Multiple cells per minicolumn = higher-order memory, allowing the same minicolumn to mean different things in different sequential contexts.
  • A single cell per minicolumn would produce first-order memory only — a chain that breaks on unexpected input with no ability to branch.

Two-Phase Algorithm Per Timestep

  • Phase 1 — Activate: For each active minicolumn, determine which cells fire. If any cell in the minicolumn was in a predictive state, only those cells fire. If none were predictive, the entire minicolumn bursts (all cells activate).
  • Phase 2 — Predict: After activation, select which cells across all minicolumns enter a predictive state for the next timestep. A cell becomes predictive if any of its basal dendritic segments has enough active presynaptic connections (above activation threshold) to the currently active cells.

Bursting and Winner Cell Selection

  • Bursting means the model has not seen this input in this context before — no cell predicted it.
  • A winner cell is selected from the bursting column to represent this new sequence step in the future.
  • Selection: prefer a cell with a dendritic segment that almost predicted the last input (permanence values below threshold but matching pattern). If no such cell exists, pick the cell with the fewest segments.
  • After winner cell selection: grow or strengthen distal synapses from the winner cell to the previously active cells (the prior context).
  • Winner cells of non-bursting columns = their active cells (already predicted correctly).
  • Include a synapse decay hyperparameter for forgetting over time.

Reference Frames, Grid Cells & Place Cells

TBT
  • Place cells (hippocampus): Represent where the animal is in an environment based on sensory input. They remap when sensory details of the environment change.
  • Grid cells (entorhinal cortex + neocortex): Represent location in a reference frame using a hexagonal tiling pattern. They do NOT remap. Fire when the sensor or organism is at specific locations in the tiled space.
  • The hexagonal tiling arises because circles packed as densely as possible produce hexagonal center patterns.
  • A single grid cell module tiles its space — the location is ambiguous across tiles. To get a unique position, multiple grid cell modules with different scales and orientations are combined. Where their firing fields simultaneously agree = the disambiguated location. More modules = more unique representable locations.
  • A unique location SDR is formed by concatenating the SDRs of each grid cell module.
  • Grid cells in the entorhinal cortex: map locations in the organism's physical environment (room-scale reference frames). Features of a room are objects within that reference frame.
  • Grid cells in the neocortex: map locations in an object's reference frame. Each cortical column's Layer 6 tracks where its sensor patch is located relative to the object it's currently touching/sensing.
  • Experimental evidence has confirmed grid cells exist in the neocortex.
  • Grid cells can represent abstract spaces — not just physical space. Human metaphors often map non-spatial concepts into spatial terms, consistent with this.

Object Reference Frames

  • When learning a new object, an origin point is chosen within the grid cell layer's reference frame. Movement through the object space updates the grid cell activations, allowing features to be associated with locations.
  • Because the location space is so vast, randomly selected origins for different objects are effectively non-overlapping — one reference frame can store many object models without collision.
  • Object representations are allocentric — defined relative to the object itself, not the sensor.

Displacement Cells

  • Location: Layer 5b (thick-tufted ET neurons). Confirmed by Hawkins 2019. Previously listed as unknown.
  • L5b has a modular structure mirroring L6a. Each L6a grid cell module is paired with one L5b displacement cell module using the same period. A single displacement cell module is ambiguous — it cannot uniquely represent a displacement, just as a single grid cell module cannot uniquely represent a position. Multiple displacement cell modules together uniquely encode a displacement via the same residue number system logic. The displacement SDR is the concatenation of all modules' outputs.
  • When L5b fires encoding displacement d, it shifts each L6a module's phase by (d mod λᵢ) via modular addition — path integration. This is the same integrate(d) call in our grid_cells.py.
  • Displacement cells serve two alternating roles: (1) movement — updating the reference frame when the sensor physically moves, and (2) composition — representing the spatial relationship between two separate object reference frames (e.g., where a logo is positioned on a cup). The same L5b population alternates between these functions.
  • Displacements allow learning new objects (traversing features during exploration) and recognizing learned objects (predicting the next feature at the next location).
  • Displacements between objects are larger than displacements between features of a single object. Large displacements move between object reference frames, enabling object composition.
  • A sequence of displacements + temporal memory = representation of an object's behavior over time.
  • Source of displacement for addition: The displacement does not have to originate in the same column. A column representing the "+" operator could fire its own L5b displacement cells and send that signal to another column's L6a. This means injecting displacement externally to L6a for the addition test is architecturally valid — it is equivalent to the displacement being provided by a separate "operator" column.
07

Object Learning & Recognition

TBT
  • A cortical column learns a complete model of an object by integrating features + locations over time as the sensor moves across the object.
  • Each cortical column builds its own independent model of every object it has experienced. Multiple columns build separate but consistent models of the same object.
  • A column for a sensor tracks the location (in its reference frame) of its associated sensory patch and processes the feedforward sensory input from that patch simultaneously.
  • Layer 4 receives the raw sensory feature (what). Layer 6 tracks the current location in the reference frame (where). Layer 2/3 stores the bound feature-location pair and is the site of inter-column voting.

Inter-Column Voting (Consensus)

  • Voting occurs at Layer 2/3. Columns whose receptive fields are spatially near each other (neighboring skin patches, neighboring retinal spots) are also physically close in the neocortex and vote with each other as neighbors.
  • If multiple columns are touching the same object (e.g. three fingers on a cup), each has a different position on the object and a different sensory experience, but all converge on a consensus object identity in one sensorimotor step.
  • A column's Layer 2/3 representation can be split and projected laterally to neighboring columns, and receives their projections, as contextual basal/distal input.
  • We are only consciously aware of the consensus output.
08

Hierarchy vs. Heterarchy

TBT
  • The classical AI view: strict feedforward hierarchy extracts features at increasing scales (edges → shapes → objects). Deep Learning depends on this model.
  • The neocortex does NOT use a strict hierarchy. More than 40% of all possible inter-region connections exist. It is better described as a heterarchy — messily entangled, wide, with bidirectional and lateral connections at all levels.
  • Where hierarchy exists in the neocortex, higher levels handle more abstract and more stable predictions (not larger feature scales). Lower levels: edges. Higher levels: object identity, meaning, behavior.
  • Hierarchy in the neocortex extracts models at different scales, not features at different scales.
  • Any level of the hierarchy can inform any other level. A local clue from a low-level column can propagate laterally or up/down freely.
  • Primary sensory regions have multimodal responses — influenced by other sensory modalities, confirming non-strict hierarchy.
  • Example: A high-level column sees the ground but not the horse. A mid-level column sees the horse but not details. A low-level column sees hair but not the animal. Combined, the system identifies the horse, its hair, and its position on the ground simultaneously.
09

Sensorimotor Integration

TBT
  • Every cortical column is engaged in its own independent sensorimotor loop. The difference between columns is only what their sensor is.
  • A column receiving retinal input also processes how the eye should move. A column receiving auditory input also directs head movement to orient the ears. Motor output is universal across all columns.
  • The reason the neocortex receives a copy of motor commands: every eye saccade produces an entirely new retinal image. Without knowing what motor command caused the change, the neocortex cannot distinguish self-caused sensory change from external change.
  • Each cortical column learns a predictive model of its receptive field: given current sensory input and the reference frame location of the sensor, predict what the sensor will observe next.
  • A visual sensory patch is not a section of the retina — it is projected across 3D space from the sensor origin. Depth perception is required for this in 3D environments.

Open Questions & Future Research

Unresolved
  • What is the role of Layer 1? Is it truly computationally irrelevant, or does it play an indispensable role?
  • What is the exact role of displacement cells — they are in L5b, but how does L5b decide which displacement to output given the current context? What learning rule governs this?
  • Some research indicates human minicolumns build group-theoretic representations. What does that mean computationally, and how should it inform the architecture?
  • What are the precise hyperparameters needed for spatial pooling? (boost factor calculation algorithm, connection threshold, active duty cycle formula, inhibition radius.)
  • Is there a faster learning method for spatial pooling than iterative Hebbian carving? Could image rescaling techniques offer an analog?
  • Where exactly does the neocortical grid cell reference frame originate? Is it truly that each cortical column's Layer 6 grid cells fire when their specific sensory patch is at the corresponding location on the object?
