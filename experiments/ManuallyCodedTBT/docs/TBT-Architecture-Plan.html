Architecture Plan // ManuallyCodedTBT

TBT Architecture Plan

HTM foundations + TBT cortical column layer model — built from scratch, no Monty

Design Principles
  • Data Oriented Design. Arrays of data, not trees of objects. Cortical columns own flat arrays. No Cell class, no Minicolumn class, no Segment class. Index arithmetic and slicing replace object hierarchies.
  • SDRs are the universal data type. Every signal between every component — sensory input, motor commands, inter-layer communication, inter-column voting — is an SDR. No floats, no embedding vectors, no 3D coordinates as primitives.
  • Reference frames are learned, not hardcoded. Grid cell modules learn the dimensionality and metric of their space from data. The system must be capable of operating in spaces that are not 3D Euclidean — semantic space, number lines, abstract relational spaces.
  • Prediction error is the only training signal. The system predicts what it will observe next. It only learns when the prediction is wrong. No backpropagation, no gradient descent, no loss functions.
  • One algorithm, every domain. The cortical column is a single repeating circuit. The same code handles vision, touch, language, and arithmetic. Domain differences come from the sensor and its encoder, not from the column.
  • Bitter lesson compliance. Emergent capability over hand-coded knowledge. The column learns representations; we do not design them. Configuration-driven architecture, not anatomy-in-code.
What Monty Got Wrong — Do Not Repeat
  • Hardcoded 3D Euclidean as the universal reference frame. All learning modules communicate in xyz coordinates. This makes non-spatial domains (language, reasoning, math) impossible.
  • Abandoned SDRs entirely. Object models are torch_geometric graphs with float features. No noise resistance, no HDC operations, no biological grounding.
  • Abandoned temporal memory. No sequence learning, no prediction-error-driven learning, no bursting.
  • No benchmarks on standard ML tasks. Only tested in a custom 3D simulation. No MNIST, CIFAR, ARC, or any compositionality benchmark.
  • The "cortical messaging protocol" forces every learning module into the same coordinate system, destroying the ability of different columns to maintain independent reference frames — which is the entire point of TBT.
01

System Architecture — Top Level

Design
  • The brain is a flat 2D sheet of regions. Each region contains cortical columns laid out according to a sensor map (retinotopic for vision, somatotopic for touch, etc.). Neighboring columns correspond to neighboring receptive fields.
  • Every sensor has a sensor map defining its receptive fields and their spatial relationships. The region that receives from that sensor mirrors this map.
  • The Old Brain (reptilian brain) sets objective functions, controls muscles, executes autonomic behavior. It receives all raw sensory data from all senses, and controls all muscles. It sends a copy of sensory data and motor commands to the neocortex — these two streams are the neocortex's only inputs.
  • The Thalamus routes sensory data to the appropriate cortical region. It is also involved in coordinate transformations between egocentric (sensor-relative) and allocentric (object-relative) reference frames. Its exact computational role is an open question.
  • All sensory data is converted into SDRs by encoders before reaching the brain. There are two encoder types: Encoder (single quantity) and MultiEncoder (semantically related quantities combined into one SDR).
  • The system runs in discrete 5ms timesteps: collect sensory data → process in brain → emit motor output → repeat.
02

Cortical Column — Layer-by-Layer Computational Model

Design
Each cortical column is a single object owning flat arrays for each layer's state. Layers are not separate objects — they are groups of arrays within the column, connected by index-based wiring. Note: L2/3 and L5 each split into two functionally distinct sublayers.

Layer 4 — Sensory Input and Feature-Location Binding

  • Role: Receives thalamic/encoder sensory input. Also receives the location prediction from L6a. L4 is where "what" (feature) and "where" (location) are first combined — binding happens here, not in L2/3.
  • Mechanism: Spatial pooling converts input SDR to stable sparse minicolumn representation. L6a's location signal modulates this processing. L4 also projects down to L6a, providing sensory grounding to the grid cell system.
  • Output: Bound feature-location SDR fed up to L3 as proximal input. Also projects to L6a.

Layer 3 (Lower L2/3) — Sequence Learning and Local Voting

  • Role: Receives feedforward input from L4. Learns sequences of feature-location pairs via temporal memory. Projects laterally to neighboring columns' L3 for inter-column voting. Projects to L5a (driving motor output). Projects feedback to L4.
  • Mechanism: Location-conditioned temporal memory — cells become predictive based on both prior sequence context and current L6a location.
  • Output: Object representation SDR fed to L5a, laterally to neighboring columns' L3, and feedback to L4.

Layer 2 (Upper L2/3) — Sensorimotor Integration and Long-Range Output

  • Role: Receives from L3 below AND from L5a. This makes L2 a sensorimotor integration layer — it combines the sensory sequence representation with the motor context. Projects strongly to other cortical areas (long-range corticocortical output).
  • Output: Integrated sensorimotor representation sent to other cortical areas.

Layer 5a — Intratelencephalic (IT) Motor Output

  • Role: IT neurons. Receives from L3. Projects to other cortical columns, other cortical areas, and the striatum — the corticocortical motor signal. Also provides input back up to L2. Receives from L6b.
  • Output: Motor context SDR to other cortical columns, striatum, and upward to L2.

Layer 5b — Displacement Cell Modules (ET Neurons)

  • Confirmed: ET neurons ("thick-tufted") are displacement cells (Hawkins 2019). L5b is NOT the same as L5a — L5a is corticocortical, L5b is displacement/motor.
  • Modular structure: L5b has one displacement cell module for each L6a grid cell module, with the same period structure. A single displacement module is ambiguous (cannot uniquely represent a displacement); multiple modules together uniquely encode a displacement via the same residue number system as L6a. Displacement SDR = concatenation of all modules' outputs.
  • Bilateral connection with L6a: Firing displacement d shifts each L6a module's phase by (d mod λᵢ) — path integration. This is the update mechanism for the reference frame.
  • Axon split: Branch 1 → subcortical motor targets (muscles). Branch 2 → thalamic relay cells → next hierarchical cortical area's L4.
  • Two roles: (1) Movement — updating L6a when the sensor moves. (2) Composition — encoding the spatial relationship between two separate object reference frames.
  • Addition test strategy: The displacement does not have to come from this column. Injecting displacement externally to L6a is architecturally valid — equivalent to a "+" operator column sending its L5b signal to our column's L6a. Phase 9 uses external injection.

Layer 6a — Grid Cells / Reference Frame

  • Role: Grid cell modules tracking sensor location in the object's allocentric reference frame. Projects a location prediction to L4. Projects modulatory feedback to the thalamic nucleus providing sensory input — modifies thalamic relay without driving it directly.
  • Mechanism: Residue number system via multiple grid cell modules with different periods. Path integration via modular phase addition.
  • Output: Location SDR to L4 (prediction) and to L3 as basal context. Modulatory signal to thalamus.

Layer 6b — Role Unclear

  • Role: Receives from L5a (per diagram). Projects to same thalamic nucleus as L6a, and also to the next higher-order thalamic nucleus. May modulate hierarchical routing. Function in adult cortex is not well understood.
  • Status: Deferred. Not implemented in initial build.

Layer 1 — Top-Down Wiring Substrate

  • Role: Contains apical dendrite tufts and long-range axons. Wiring substrate for top-down feedback. Function implicit in connections defined elsewhere. Not modeled separately.

Phase 10 — Learned L5b and Reference Frame Fix

Next

Phase 10 has two coupled components that must be implemented together because they share a root cause: the location signal is currently too weak to independently activate segments.

Component A — Fix: Location-Driven Predictions (No More Reset Burst)

Root problem: After reset, prev_active is all zeros, so cell synapses contribute nothing to segment activity. Location synapses are additive and rarely reach threshold alone. The column always bursts at step 0 of any new context, even for positions it has been to thousands of times.

This is architecturally wrong. In a biological column, the location signal arrives on basal/distal dendrites and independently puts cells in a depolarized "ready to fire" state. When sensory input then arrives on proximal dendrites, the pre-primed cells fire selectively without any sequential context being required. The reference frame must be position-addressable, not walk-dependent.

The correct behavior: when the grid layer is at position p, the cells associated with feature(p) at that position should become predictive from the location signal alone, and when feature(p) arrives they fire without bursting — even after a reset.

This also resolves the multi-stride training problem from Phase 9a. If location alone can make the correct cells predictive, there is no burst-winner competition between strides at shared positions. Training one stride no longer destabilizes another.

Implementation: Add a location_activation_threshold parameter to TemporalMemory (separate from segment_activation_threshold). A segment becomes predictive if either:

  • Combined (cell + location) synapses ≥ segment_activation_threshold (existing behavior), OR
  • Location synapses alone ≥ location_activation_threshold (new — location-only activation)

This is backward-compatible — setting location_activation_threshold above max_loc_synapses_per_seg restores the old additive-only behavior. Growing enough location synapses per segment (12–16) and setting threshold = 8–10 should let mature segments fire from location alone.

Biological grounding: This maps directly onto the known distinction between proximal (driving) and basal/distal (modulatory) dendritic input. Distal dendritic input can independently depolarize a pyramidal cell's apical tuft, lowering the threshold for proximal input to cause firing. In HTM terms, the location SDR on basal dendrites acts as a dendritic spike threshold, not just an additive vote.

Component B — Learned L5b Displacement Cells

The problem 9b demonstrates: without a learned L5 displacement output, the column cannot learn that "+" means "treat the next number as a displacement." It can only rote-memorize specific (a, b) → result pairs, and fails on any pair where multiple inputs share an output.

Architecture (from Hawkins 2019 + pre-compaction design):

  • L3 → L5a (conjunctive readout): L5a (IT neurons) receives from L3 (confirmed) and from L4. L5a acts as the conjunctive integrator of operator context (from L3: "we are after +") and current feature magnitude (from L4: "the number is 5"). L5a is where the learned displacement computation lives, not L5b.
  • L5a → L5b (local drive): L5a drives L5b (ET displacement cells) locally. L5b produces the actual displacement output to L6a and subcortical motor targets. L3 does NOT directly drive L5b — this is confirmed by Hawkins 2019.
  • L5b → L6a (path integration): L5b fires displacement d, shifting each L6a module's phase by d mod λᵢ — the same grid_layer.integrate(d) call already implemented.

The learned mapping: L5a learns a weight matrix W of shape (total_L3_cells + num_minicolumns, 1) mapping (L3 active cells ∥ L4 active minicolumns) → displacement scalar. This is a conjunctive readout: the same L4 input (number 5) produces zero displacement in neutral context and displacement +5 in "after +" context, because L3 context changes which cells contribute to the dot product.

Learning rule (Hebbian, prediction-error-driven):

  1. At each timestep, L5a outputs displacement d̂ = W · (L3_cells ∥ L4_cols).
  2. d̂ is applied to L6a. The column then observes the next sensory input.
  3. If the prediction was correct (anomaly = 0 at the next step), reinforce: strengthen synapses from currently-active L3 cells and L4 minicolumns toward d̂.
  4. If prediction was wrong (anomaly > 0), weaken those synapses.
  5. The anomaly score at the RESULT step (after "=") is the primary teaching signal.

This is Option B from the Phase 9b test output. It avoids backpropagation. It is biologically plausible — the anomaly signal corresponds to prediction error, which is already computed by the TM and propagated as bursting. No external supervision is needed.

Implementation plan:

  • New file: l5a_readout.py — L5aReadout class with weight matrix W, compute(l3_cells, l4_cols) → displacement scalar, learn(displacement, anomaly_score) for Hebbian update.
  • L5b is already implemented as DisplacementLayer. L5a feeds into it via displ.apply_displacement_to(d, grid_layer).
  • CorticalColumn updated: compute() accepts an optional L5aReadout; if provided, it runs the readout after L3 TM and before applying displacement to L6a.
  • Training sequences: [a, +, b, =, a+b] — no external displacement injection. L5a must learn to output b when L3 context = "after +" and L4 = feature(b).

Test Plan (test_addition_10.py)

  • Test A — Reset no longer bursts at known positions: Train the column on stride-1 walk. Reset and jump directly to position 5 (set_position). Observe feature(5). Anomaly should be near 0.0. This validates Component A.
  • Test B — Multi-stride no longer requires isolated training: Train stride-1 and stride-2 simultaneously. Both should converge in ≤500 epochs. This validates that Component A resolves the training dynamics problem from Phase 9a.
  • Test C — Unique-result addition with learned L5a: Train on [a, +, b, =, a+b] sequences without displacement injection. Measure anomaly at result step for unique-result pairs. Expected to pass (previously passed even without L5b via rote memorization, so this is a sanity check).
  • Test D — Ambiguous-result addition with learned L5a: Test pairs where multiple (a,b) share the same result (1+9=10 and 5+5=10). This is the critical test. With L5a providing the correct displacement, the reference frame should be at position a+b at the result step, so feature(a+b) is predicted correctly via the reference frame rather than via rote sequence memory. Expected to PASS for the first time.
  • Test E — Generalization: Test pairs not seen during training. With reference frame generalization, unseen (a,b) pairs should work as long as positions a and a+b were visited.

Open Questions for Phase 10

  • What learning rate and decay schedule works best for the L5a weight matrix given the sparse SDR inputs?
  • Should L5a output a continuous scalar displacement or a quantized/modular one? Continuous is simpler but the displacement layer operates on scalar values anyway, so this may not matter.
  • How does L5a know to output 0 (no displacement) when the context is neutral — i.e. when L3 is NOT in "after +" state? The weight matrix should learn near-zero weights from neutral-context L3 cells, but this needs to be verified.
  • The teaching signal (anomaly at the result step) is delayed by one step from when L5a fires. How does the system associate the error with the L5a weights from the prior step? This requires storing the prior (L3, L4) state for credit assignment.

Token Semantics — Language Modeling Notes

Future
  • For language modeling: a fixed token vocabulary, but instead of a tokenizer, a token encoder converts tokens into SDRs.
  • In transformers, embedding vectors cluster semantically similar tokens together in high-dimensional space, with similarity measured by cosine distance. SDRs can do the same — semantically similar tokens should have high bit overlap.
  • To achieve this, the SDR encoding for tokens must have a topology corresponding to semantic space. Semantically similar tokens should produce SDRs with high overlap.
  • We cannot know where every token belongs in semantic space a priori. This has to be learned — likely through the spatial pooler and/or a dedicated semantic learning mechanism.
  • The dimensionality of semantic space is NOT fixed at 3. It is potentially very high-dimensional. Grid cell modules must support this — they project onto an abstract space, not necessarily physical space.

Open Questions

Unresolved

Grid Cells & Reference Frames

  • How do grid cells decide how to project themselves onto a space? How do they select a metric and dimensionality? Is this configured per-region, or learned?
  • How does L6a's location prediction to L4 work mechanistically? Is it a gating signal, an additive modulatory input, or something else entirely?

Displacement Cells and Layer 5b

  • L5b displacement cells are confirmed (Hawkins 2019) and their modular structure is now understood. The remaining question: what learning rule governs L5b's displacement output? Given operator context "after +" and current L4 input (the number 5), how does L5b learn to output displacement +5?
  • Does the Old Brain's goal/reward signal directly modulate L5b, biasing which displacements get reinforced?

Layer 6b

  • What does L6b actually compute? It receives from L5a and projects to the same thalamic nucleus as L6a plus the next higher-order thalamic nucleus. Is it routing decisions for the hierarchical chain? Its function in adult cortex is genuinely unresolved in the neuroscience literature.

Layer 2 vs Layer 3

  • L3 does the sequence learning and local voting. L2 integrates L3 output with L5a motor context and projects long-range. But what specifically is L2 computing — is it a consensus object representation with motor context baked in, or something else?
  • When L5a projects back up to L2, what does that motor context signal represent at the L2 level?

Inter-Column Communication

  • L3 → neighboring columns' L3 for local voting. Which layers do L5a project to in receiving columns — L3, L5, both?
  • How does inter-column voting work computationally? Is it SDR union of object hypotheses with overlap determining consensus?
  • How does long-range L2 → other areas work? Which layers does it target in the receiving area?

Thalamus

  • L6a → thalamus is modulatory (modifies how relay cells respond without driving them). L6b also projects to the next higher-order thalamus. How does this thalamic routing interact with the cortical hierarchy — is L6b the mechanism for "level skipping" that Hawkins describes?
  • The thalamus receives a copy of L5b's motor output and relays it to the next cortical area's L4. What transformation, if any, does the thalamus perform on this signal?

Biological Open Questions

  • Some research indicates minicolumns build group-theoretic representations. What does this mean computationally? Could it inform how L6a grid cells tile their spaces?

Milestone Test — Addition & Subtraction on the Number Line

Plan
Core Insight

Addition and subtraction are navigation on a 1D number line. "3 + 5 = 8" means: start at position 3, apply displacement +5, arrive at position 8. A cortical column with a 1D reference frame should be able to learn this as a sensorimotor pattern — no different from learning that moving your finger 2cm along a cup brings you from the handle to the rim.

  • Sensor: A number encoder. Encodes integers into SDRs using scalar encoding on a number line. The "sensory input" is the current number being observed.
  • Reference frame: A 1D grid cell layer. Position on the number line. Displacement = the addend or subtrahend.
  • Object model: The number line itself. Feature at location N = the SDR encoding of N. The column learns that every position on the number line has a feature (the number at that position), and that displacements move between positions.
  • The task: Given input "3 + 5", the column receives the SDR for 3, applies displacement +5 to its reference frame, and predicts the feature at the new location. If it has learned the number line, it predicts the SDR for 8.
  • Generalization test: Train on a subset of additions (e.g., all pairs where both operands are 0–10). Test on held-out pairs (e.g., operands 11–20). If the column has truly learned the number line as a reference frame — not memorized input-output pairs — it should generalize perfectly.
