Cortical Column — Layer Design & Wiring

Mathematical theory, neuroscience, and implementation design for each cortical layer

I

Mathematical Theory of the Brain — Broad Principles

Math

I.1 — Symmetry-Based Representations

A symmetry transformation is one that changes some aspect of a system while leaving something else invariant. A rotation changes an object's orientation but preserves its shape. A translation changes position but preserves distances. A good neural representation is one that reflects the symmetry structure of the domain it operates in — it is equivariant to the transformations that matter and invariant to those that don't [Higgins2022].

Group theory is the mathematical language of symmetry. A group (G, ∘) is a set G with a binary operation ∘ satisfying closure, associativity, identity, and invertibility. Every domain the brain navigates — physical space, the number line, semantic concept space — has a symmetry group describing the transformations that preserve its structure.

The hypothesis: the brain's neural population activity preserves the essential mathematical structures (group, metric, topology) of the tasks it supports. This is the Principle of Isomorphism (PIso) proposed by Gao et al. — population activity is a group representation of the task's symmetry group [Gao2024].
  • Physical 2D space has the symmetry group of translations and rotations: the Euclidean group E(2). Grid cells in the entorhinal cortex implement a representation of this group — their firing patterns are equivariant to translation and rotation of the animal.
  • The number line ℝ under addition has the symmetry group (ℝ, +) — the additive group of real numbers. A 1D grid cell system implements this by performing modular addition on phase states.
  • An abstract concept space (e.g. car-space spanned by engine power × weight, or a social hierarchy) has its own symmetry structure. Grid-like codes have been observed in entorhinal cortex and vmPFC during navigation of such abstract spaces [Park2021, Bao2019, Bellmund2018].
  • The same grid cell mechanism applies regardless of the space's dimensionality or whether it is physical or abstract. This is why TBT claims neocortical grid cells do the same thing as entorhinal grid cells, but for object-relative reference frames rather than environment-relative ones.

I.2 — Residue Number Systems and the Grid Code

A residue number system (RNS) represents a number by its remainders (residues) under a set of moduli. If the moduli are coprime, the Chinese Remainder Theorem guarantees that the representation is unique for all numbers in the range [0, M) where M is the product of the moduli [CRT].

Number x  ↔  (x mod λ₁, x mod λ₂, ..., x mod λₖ)

Unique for x ∈ [0, λ₁ × λ₂ × ... × λₖ) if gcd(λᵢ, λⱼ) = 1 for all i ≠ j

The critical property: addition, subtraction, and multiplication in an RNS decompose into independent modular operations on each residue [Sreenivasan2011]. You don't need to carry. Each module operates in parallel, independently:

(x + d) mod λᵢ  =  (x mod λᵢ + d mod λᵢ) mod λᵢ    for each module i independently

Grid cells implement exactly this. Each grid cell module has a spatial period λᵢ. The module's population activity encodes position mod λᵢ (the phase). Multiple modules with different periods form a residue number system. Path integration — applying a displacement d — is modular addition on each module's phase, performed independently and in parallel [Sreenivasan2011, Fiete2008].

This is why grid cells can represent positions over a range exponentially larger than any single module's period. With k modules having periods λ₁...λₖ, the representable range is the product λ₁ × λ₂ × ... × λₖ, but each module only needs ~λᵢ/δ distinguishable states (where δ is the spatial resolution). The capacity scales exponentially with the number of modules while the state space grows only linearly.

I.3 — Group Representations in Grid Cell Populations

Recent theoretical work [Gao2024] formalizes what grid cells compute as a group representation. Path integration preserves the algebraic structure of displacements:

  • Displacements in 2D space form the additive group (ℝ², +). Grid cell population activity provides a group homomorphism from this group to the group of neural state transitions — applying displacement d₁ then d₂ produces the same neural state as applying d₁+d₂ directly.
  • This homomorphism is the formal definition of what it means for the brain to "preserve the algebra of displacement" — the neural system is a faithful representation of the displacement group.
  • The commutativity of displacement summation (d₁ + d₂ = d₂ + d₁) is preserved: grid cell path integration gives the same result regardless of the order displacements are applied. This has been experimentally verified.
  • For 1D (the number line): the group is (ℝ, +), which is Abelian (commutative). Each grid cell module implements a representation of the quotient group ℝ/λℤ (the circle group of circumference λ). The product of all modules represents the full group up to the system's range.

I.4 — Information Field Theory Connection

Information Field Theory (IFT) recasts Bayesian inference over continuous fields using statistical physics machinery. The connection to cortical columns:

  • The prior encodes structural assumptions about the field — smoothness, symmetries, local correlations. In the cortical column, the spatial pooler's learned permanence structure acts as a prior: it has learned which input patterns are "comfortable" (low energy / high probability) configurations.
  • The likelihood models how noisy measurements relate to the true field. Layer 4's sensory input is a noisy measurement of the world. The spatial pooler transforms this noisy input into a stable internal representation.
  • The posterior — the best reconstruction given data and prior — corresponds to the cortical column's settled state after integrating sensory input (L4), location context (L6), and temporal context (L2/3 basal).
  • Bursting in temporal memory is an expression of high posterior uncertainty — the system is saying "multiple field configurations are equally plausible given the current data."
  • The renormalization group principle — that good representations keep task-relevant structure and discard noise — maps onto TBT's claim that hierarchy extracts models at different scales, not features at different scales. Each level is a coarse-graining.

I.5 — Implications for Learning Rules vs. Learning Content

The group-theoretic view clarifies the separation between what must be learned and what is computed structurally:

  • Not learned (structural): The group operation itself. If the grid cell modules are correctly wired to perform modular phase shifts in response to displacement inputs, then addition/subtraction/path-integration is performed natively by the reference frame. The column does not learn that 3+5=8 — it computes it by applying displacement +5 to position 3.
  • Learned (by temporal memory / L2/3): Context-dependent routing rules. The system must learn that the symbol "+" means "interpret the next sensory input as a displacement to apply to the reference frame." This is sequence learning — the same machinery HTM already has. The temporal memory learns that in the context [number, +], the next input should be routed to L6 as a displacement rather than processed as a feature at the current position.
  • Learned (by feature-location binding / L2/3): The mapping between positions in the reference frame and the features that occur at those positions. On the number line, this means learning that position 8 corresponds to the sensory encoding of the number 8. This is the "object model" — the number line is the object, and each position on it has a feature (the number at that position).
  • Learned or configured (L4 → L6 pathway): How to convert a sensory number encoding into a displacement magnitude. The SDR for "5" (a sensory feature) must be translated into a displacement of magnitude 5 in the L6 reference frame. This mapping is the crucial learned connection between the "what" and the "how far."
Key Insight

Compositionality — the ability to combine known parts in new ways — comes from the reference frame's algebraic structure, not from memorizing combinations. If the grid cells correctly implement the group (ℝ, +) via a residue number system, then any displacement applied to any position gives the correct result, even for operand pairs never seen during training. Generalization is a structural property of the representation, not a statistical property of the training data.

II.6

Layer 6 — Grid Cell / Reference Frame Layer

Design + Math + Neuro

Biological Role

  • Layer 6 tracks where the sensor is in the object's allocentric reference frame. This is the "where" of the cortical column.
  • Contains multiple grid cell modules with different spatial periods (scales) and orientations. Their concatenated activity forms a unique location SDR.
  • Receives displacement signals (motor copy / movement commands) and performs path integration — updating the location estimate by applying the displacement to the current grid cell state.
  • Grid cells in the entorhinal cortex encode environment-scale positions. Grid cells in the neocortex (Layer 6 of cortical columns) encode object-scale positions — where the sensor is relative to the object being sensed [Hawkins2019].
  • Grid cell modules are organized along the dorsoventral axis of the MEC, with spatial period increasing systematically [Stensola2012]. The scale ratio between consecutive modules appears to be consistent across animals (~1.4–1.7), suggesting a genetic component [Stensola2012].
  • Grid representations are not limited to navigational space — grid maps of visual space, value space, and abstract conceptual spaces have been demonstrated [Killian2012, Constantinescu2016, Park2021].

Mathematical Theory

Grid Cell Module — Phase Representation

A single grid cell module with period λ represents position as a phase φ ∈ [0, λ). In 1D, this is simply x mod λ. In 2D, the module tiles the plane with a hexagonal lattice characterized by period λ and orientation θ. The module's population activity is a function of the phase:

1D module:   φᵢ(x) = x mod λᵢ

2D module:   φᵢ(x, y) = (x, y) projected onto the module's lattice basis vectors, then mod λᵢ

Phase update (path integration):   φᵢ ← (φᵢ + d) mod λᵢ   where d is the displacement

Multi-Module Residue Number System

With k modules having periods λ₁, λ₂, ..., λₖ, the system represents position as a vector of phases:

Position x  →  (x mod λ₁, x mod λ₂, ..., x mod λₖ)

Unique range: [0, lcm(λ₁, λ₂, ..., λₖ))

If periods are coprime: unique range = λ₁ × λ₂ × ... × λₖ

Addition (path integration) decomposes into independent per-module operations:

Position(x + d) = ( (x mod λ₁ + d mod λ₁) mod λ₁,
                    (x mod λ₂ + d mod λ₂) mod λ₂,
                    ...
                    (x mod λₖ + d mod λₖ) mod λₖ )

This is the carry-free property of residue number systems. No module needs to communicate with any other module during path integration. Each operates independently and in parallel.

SDR Encoding of Phase

Each module's phase must be encoded as an SDR for compatibility with the rest of the column. A natural encoding: use periodic scalar encoding (from sdr.py) with the module's period as the range. The phase φ ∈ [0, λ) maps to a periodic SDR where min_val=0, max_val=λ.

The full location SDR is the concatenation of all modules' phase SDRs:

location_sdr = concat(encode_periodic(φ₁, n₁, w₁, 0, λ₁),
                     encode_periodic(φ₂, n₂, w₂, 0, λ₂),
                     ...
                     encode_periodic(φₖ, nₖ, wₖ, 0, λₖ))

Because each sub-SDR uses periodic encoding, the phase wraps correctly — position λ has the same encoding as position 0 within that module, which is exactly the cyclic group ℤ/λℤ.

Error Correction

The residue number system provides intrinsic error correction [Sreenivasan2011]. Not all combinations of phases are valid — in a system with coprime periods, only 1/M of all possible phase combinations correspond to actual positions (where M is the product of periods). If noise corrupts a module's phase, the invalid combination can be detected and the closest valid position recovered. This is analogous to error-correcting codes and is a major advantage over a simple place-cell encoding.

Choosing Periods

  • For maximum unique range with minimum modules: choose coprime periods. Example: λ = {7, 11, 13} gives a unique range of 1001 with only 3 modules.
  • Biological evidence suggests a geometric progression of scales (each ~1.4–1.7× the previous) rather than coprime integers [Stensola2012]. This sacrifices some range but provides better noise robustness [Fiete2008].
  • For the 1D number line (addition test): start with coprime integer periods, e.g. {7, 11, 13}, giving a range of 0–1000. This is far more than needed for testing addition of small numbers, which provides a margin for generalization testing.

Displacement Cells — Located in L5b, Not L6

  • Location confirmed: Displacement cells are in Layer 5b (thick-tufted ET neurons), not in L6. This was previously unknown. Source: Hawkins 2019.
  • Structure: L5b has one displacement cell module paired with each L6a grid cell module, using the same period. The pairing is 1:1. A displacement d is encoded as a vector of (d mod λ₁, d mod λ₂, ..., d mod λₖ) — the same residue number system as L6a position encoding, but representing differences between positions rather than positions themselves.
  • Path integration: When L5b fires encoding displacement d, each paired L6a module's phase updates by (d mod λᵢ) — modular addition. This is exactly the grid_layer.integrate(d) call in our implementation.
  • Negative displacements (subtraction): In modular arithmetic, subtraction is addition of the additive inverse. In ℤ/λℤ, the inverse of d is λ − d. So subtracting 3 in a module with period 7 is the same as adding 4. The system handles subtraction natively — no special case needed.
  • Two roles: (1) Movement — updating L6a when the sensor physically moves. (2) Composition — encoding the displacement between two separate object reference frames.

Inputs and Outputs

  • Input to L6a — from L4 (sensory grounding): L4 projects down to L6a. When L4 receives a known sensory feature, this allows L6a to correct its grid cell phases to match the stored location of that feature — resetting path integration drift.
  • Input to L6b — from L5a (per diagram): L5a's corticocortical output also projects to L6b. The computation L6b performs with this input is not yet understood.
  • Output — L6a → L4 (location prediction, modulatory): Concatenated grid cell phase SDRs sent upward to L4 as a predictive/modulatory signal. Combined with orientation, this is how the column predicts what sensory input to expect given current position. This does NOT drive L4 cells to fire — it modulates their response to incoming sensory input.
  • Output — L6a → L3 (location context, basal): Location SDR provides basal/distal context to L3 temporal memory, enabling feature-location binding in sequence learning.
  • Output — L6a → Thalamus (modulatory feedback): L6a sends feedback to the thalamic nucleus providing sensory input to this column. This is modulatory only — it modifies how thalamic relay cells respond to subcortical input but does NOT cause them to fire directly.
  • Output — L6b → Thalamus (routing): L6b projects to the same thalamic nucleus as L6a, and also to the next higher-order thalamic nucleus. This may be the routing mechanism for the hierarchical chain. Function not fully understood.

Sublayer Structure

  • L6a — Grid cells: Contains the grid cell modules. Implements the residue number system for position. Projects location prediction to L4. Sends modulatory (not driving) feedback to the thalamus. Receives sensory grounding from L4.
  • L6b — Role unclear: Projects to the same thalamic nucleus as L6a, plus to the next higher-order thalamic nucleus. Receives from L5a. May be a subplate remnant. Function in adult cortex is actively under investigation and is not well understood. L6b is deferred in our implementation.

Open Questions for Layer 6

  • How does the system learn the appropriate periods for a new domain? Are they fixed hyperparameters, or learned from data?
  • How does L6a's location prediction to L4 work mechanistically — is it concatenated to encoder input, used as a gating mask, or something else?
  • What does L6b compute from its L5a input, and how does its thalamic routing function?
  • In 2D and higher dimensions, how are lattice orientation and basis vectors determined for each grid cell module?
  • How does sensory anchoring work — when L4 recognizes a known feature, what mechanism corrects L6a's phases to the expected location?
II.4

Layer 4 — Sensory Input and Feature-Location Binding

Design

Biological Role

  • Layer 4 is the primary recipient of thalamic sensory relay input. Feedforward sensory data arrives here first.
  • Layer 4 also receives the location prediction from L6a — making it the site where "what" (sensory feature) and "where" (object-relative location from grid cells) are first combined. This resolves a puzzle: L3 connects to L4 but not to L6a directly. L3 does not need to reach L6a because L4 has already performed the binding before projecting upward to L3.
  • L4 projects upward to L3, carrying the bound feature-location representation. L4 also projects downward to L6a, providing sensory grounding to the grid cell reference frame (enabling sensory anchoring / phase correction).

Mathematical Theory

  • Spatial pooling as a learned prior over input configurations. Takes the input SDR and maps it to a fixed-size minicolumn representation preserving semantic overlap.
  • The L6a location signal acts as context for this pooling — the system can learn location-dependent feature representations (the same physical feature maps to a different minicolumn pattern depending on where the sensor is).
  • Hebbian learning with boosting as an energy-based homeostatic model.

Implementation Status

  • Spatial pooler implemented and tested in spatial_pooler.py.
  • L6a → L4 modulatory connection not yet implemented. Currently L4 receives only encoder output. Adding location modulation is a future step.

Open Questions

  • How does L6a's location signal modulate L4 spatially? Does it gate certain minicolumns, shift permanence thresholds, or act as an additional input concatenated to the encoder output?
  • How does L4 → L6a feedback enable sensory anchoring — when the system recognizes a known feature, how does it use this to correct grid cell phases?
II.3

Layer 3 (Lower L2/3) — Sequence Learning and Local Voting

Design

Biological Role

  • Receives feedforward proximal input from L4 — the bound feature-location representation produced by L4's combination of sensory input and L6a location signal.
  • Projects to all layers of the home column, including a feedback projection back to L4.
  • Projects laterally to neighboring columns' L3 — this is the inter-column voting pathway for object consensus within the same cortical area.
  • Projects downward to L5a, driving the corticocortical motor output.
  • Research note: lower L2/3 (L3) transcolumnar projections are relatively sparse compared to upper L2/3 (L2), which projects more strongly to other areas.

Computational Role

  • Temporal memory operates here. L3 learns sequences of bound feature-location pairs as they arrive from L4. Cells within minicolumns encode which sequence context is active, enabling the same minicolumn to represent different things in different temporal contexts.
  • The location SDR from L6a provides basal/distal context to L3 cells, so predictions are conditioned on both prior sequence context AND current position. This is implemented via the location synapses in our temporal memory.
  • Lateral projections to neighboring columns' L3 carry the current object representation SDR, enabling columns to vote with each other on object identity.

Implementation Status

  • Location-conditioned temporal memory implemented in temporal_memory.py. Accepts L6a location SDR as basal context via separate location synapse arrays on each dendritic segment.
  • Inter-column lateral voting not yet implemented — deferred until multi-column experiments begin.

Open Questions

  • When L3 → L5a drives motor output, is it the raw active cell SDR or a processed/pooled version that L5a receives?
  • How exactly does L3 → L4 feedback work? Is it a prediction of the next expected L4 state, or does it modulate L4's processing of the current input?
  • How does inter-column voting work at scale? SDR union + overlap threshold is the working hypothesis.

II.5

Layer 5 — Motor Output (L5a: IT Corticocortical / L5b: ET Displacement Modules)

Design

L5a — Intratelencephalic (IT) Neurons — Corticocortical Output

  • Cell type: IT neurons — small cell bodies in upper L5. Thin apical dendrite.
  • Inputs: Receives from L3 (driving input). Receives from L6b.
  • Outputs: Projects to other cortical columns within the same area. Projects to other cortical areas. Projects to the striatum (basal ganglia). Projects back up to L2 (upper L2/3), contributing the motor context that L2 integrates with L3's sensory sequence.
  • Role: Corticocortical and corticostriatal output — the pathway by which one column shares its current sequence state with other columns and the basal ganglia. L5a is NOT the displacement layer. L5a does not drive muscles.

L5b — Extratelencephalic (ET) Neurons — Displacement Cell Modules

  • Cell type: ET neurons — large "thick-tufted" cell bodies in lower L5. Extensive apical dendrite arborizing in L1. Confirmed as displacement cells by Hawkins 2019.
  • Modular structure: L5b has one displacement cell module for each L6a grid cell module, with the same period structure. This mirrors L6a's residue number system architecture exactly, but for displacements rather than positions. A single displacement cell module is ambiguous — it cannot uniquely represent a displacement, just as a single grid cell module cannot uniquely represent a position. Multiple displacement modules together uniquely encode a displacement. The displacement SDR is the concatenation of all modules' phase outputs.
  • Bilateral connection with L6a: L5b displacement cell modules connect bidirectionally with their paired L6a grid cell modules. When L5b fires encoding displacement d, each L6a module's phase shifts by (d mod λᵢ) — path integration. This is the update: grid_layer.integrate(d).
  • Axon split: Branch 1 → subcortical motor targets (brainstem, spinal cord, superior colliculus) — drives muscles. Branch 2 → thalamic relay cells → next hierarchical cortical area's L4 — drives hierarchical processing.
  • Two alternating roles: (1) Movement — updating L6a reference frame phases when the sensor physically moves. (2) Composition — encoding the spatial relationship between two separate object reference frames (e.g., the displacement from a cup's reference frame to a logo's reference frame). The same L5b population alternates between these two functions depending on context.
  • L4 prediction: Combined with an orientation signal, L5b displacement cells also project to L4 — this is how the column predicts what sensory input to expect next given the current displacement.

Key Clarifications

L3 → L5a is the confirmed projection (IT neurons). L3 does NOT directly drive L5b (ET neurons). L5b receives from L5a locally. The motor output to muscles (L5b) is not driven directly by L3's sequence representation — it goes through L5a first.
For the addition test: the displacement does not have to come from this column. Injecting displacement externally to L6a is architecturally valid — equivalent to a "+" operator column sending its L5b signal to this column's L6a. Phase 9 used external displacement injection. Phase 10 implements learned L5a.

Phase 10 Design — L5a Conjunctive Readout

L5a acts as a conjunctive readout of (L3 operator context, L4 current feature) → displacement scalar:

  • Inputs: L3 active cells (operator context: "we are after +") and L4 active minicolumns (current feature: "the number is 5"). Neither alone is sufficient — the same L4 input (number 5) should produce zero displacement in neutral context and displacement +5 in "after +" context.
  • Weight matrix W: Shape (total_L3_cells + num_minicolumns, 1). Maps (L3 cells ∥ L4 minicolumns) → scalar displacement. Implemented as l5a_readout.py.
  • Learning rule (Hebbian, prediction-error-driven): L5a outputs displacement d̂ which is applied to L6a. At the result step, the anomaly score is the teaching signal. If anomaly = 0 (correct), reinforce W from the stored prior (L3, L4) state. If anomaly > 0 (wrong), weaken those weights. No backpropagation required.
  • Credit assignment: The prior (L3, L4) state must be stored for one step, since the error signal at the result step refers to the L5a output from the b-number step.
  • L5a → L5b: L5a's computed displacement drives L5b (DisplacementLayer) via displ.apply_displacement_to(d_hat, grid_layer).

Open Questions

  • What learning rate and decay schedule works best for W given sparse SDR inputs?
  • Should L5a output a continuous scalar or quantized displacement? (Probably continuous — DisplacementLayer accepts a float.)
  • How does L5a learn to output 0 in neutral context? The weight matrix should naturally learn near-zero weights from neutral-context L3 cells, but needs verification.
  • Does the Old Brain's goal/reward signal directly modulate L5b, biasing which displacements get reinforced?

II.8

Phase 8 — Column Assembly Implementation Notes

Implemented

Design: Wiring Harness, Not Factory

  • The column (cortical_column.py) accepts pre-built components — encoder, SpatialPooler, TemporalMemory, GridCellLayer, DisplacementLayer — and wires them together. It does not build components internally, except as defaults when none are provided.
  • This is architecturally correct: in the brain, different cortical columns serving different sensors and modalities have different encoders and different grid metrics. The column itself is generic — the components define its domain.
  • Encoders are optional. If none is provided, compute() accepts a pre-encoded boolean SDR directly. This allows testing without an encoder and supports non-scalar input types.
  • DisplacementLayer is auto-created paired to the GridCellLayer if not provided, using make_displacement_layer_from_grid(). This guarantees period compatibility.
  • The TM's location_sdr_length is auto-detected from the GridCellLayer's output length if not specified in tm_kwargs.

Per-Timestep Computation Order

Encode → L4 (SP) → L6a location → L3 (TM with location context) → L5b displacement (if provided)
  • Step 1 — Encode: Raw sensory value → input SDR via encoder. If input is already an SDR, skip.
  • Step 2 — L4 Spatial Pooler: input_sdr → active_minicolumns. Hebbian learning updates permanences if learn=True.
  • Step 3 — L6a location: grid_layer.get_location_sdr() called BEFORE displacement is applied, so TM sees the location that was active when the current feature arrived — the binding is (current feature, current position).
  • Step 4 — L3 Temporal Memory: (active_minicolumns, location_sdr) → active_cells. Sequence learning updates segments if learn=True.
  • Step 5 — L5b displacement: If a displacement float is provided, displacement_layer.apply_displacement_to(d, grid_layer) updates L6a's phases for the next timestep. Displacement is applied AFTER observation, so the next compute() sees the updated position.

Location Conditioning Timing

The location SDR used for L3 binding is the grid layer's state BEFORE any displacement is applied this step. This correctly represents "what position was the sensor at when it observed this feature" — the binding that enables object recognition.

Performance Optimization

  • The TM's _compute_segment_activity was optimized to skip empty segments. It now collects only the indices of segments that have actually been created (cell_num_segs > 0) and operates only on those, rather than iterating over the full pre-allocated (total_cells × max_segs_per_cell) matrix. This gives ~10× speedup in early training and maintains performance as segment count grows.

Validated Capabilities (27 tests passing)

  • Anomaly score = 1.0 on first input (everything bursts — no prior context).
  • Anomaly drops below 0.5 after training on a 5-element sequence (40 epochs).
  • Location conditioning: location synapses grow during training; feature predicted correctly after preamble at a specific position.
  • Path integration: position 3 + displacement 5 = position 8 correctly.
  • Reset clears TM state but preserves SP permanences and grid phases.
  • Higher-order memory: orders 2–6 all achieve 0.000 anomaly on the end element of two sequences sharing an N-2 element middle. The column correctly disambiguates based on the first element across 6 steps of context.

What Is Not Yet Implemented

  • L6a → L4 modulatory connection (location prediction to spatial pooler).
  • L2 (upper L2/3) as a distinct layer from L3.
  • L5a corticocortical output.
  • L6b and its thalamic routing.
  • Inter-column voting (L3 lateral projections).
  • Learned L5b displacement output — displacement is currently provided externally.
II.10

Phase 9 Findings and Phase 10 Implementation Plan

Next

Phase 9a Finding — Walk-Based Training Required

  • Isolated 2-step episodes (reset → observe feature(a) → displace → observe feature(a+d)) fail to converge because after every reset, the TM bursts and selects winner cells based on current cell_num_segs, which drifts across epochs. Winner cells from epoch N don't match winner cells from epoch N+1.
  • Continuous walks (reset once, observe positions 0,1,2,...,10 without interruption) converge cleanly because every step past the first is in predicted mode — deterministic and stable.
  • Stride-1 walk trained in isolation: 0.000 anomaly at every step, 1.000 anomaly on wrong features. Perfect.
  • Multi-stride training (stride-1 and stride-2 simultaneously) fails in the same epoch budget because both strides burst at their shared starting positions, competing for winner cell slots. Requires O(1000+) epochs per competing stride. This is addressed by the Phase 10 reset burst fix.

Phase 9b Finding — Rote vs Compositional Addition

  • Without a learned L5 displacement, the column rote-memorizes specific (a,b)→result sequences via temporal memory.
  • Unique-result pairs (3+5=8, 2+7=9, 0+8=8): 0.000 anomaly. The TM memorized these exactly.
  • Ambiguous-result pairs (1+9=10 and 5+5=10 share a result): ~1.000 anomaly. The column cannot distinguish these without the reference frame being at position 10 for both — but since no displacement was applied, the grid stayed at position 0 for all sequences.
  • The failure mode is architecturally clean: it demonstrates precisely that the reference frame is what enables compositional addition. The test correctly identifies the missing component.

Phase 10, Part A — Fix: Location-Driven Prediction (Reset Burst)

Root cause of reset burst: After col.reset(), prev_active is all zeros. Cell synapses contribute nothing to segment activity. Location synapses are additive and rarely reach segment_activation_threshold alone. Result: every first step after a reset bursts, even for well-trained positions.

Why this is wrong biologically: In the neocortex, the location signal (from L6a, arriving on basal/distal dendrites) can independently depolarize a pyramidal cell — a dendritic spike — that primes the cell to fire when proximal input arrives. The reference frame must be position-addressable: placing the grid at position p should immediately prime the correct cells to predict feature(p), without requiring a walk to build sequential context.

Fix: Add location_activation_threshold (int) to TemporalMemory. A segment becomes predictive if EITHER:

  • Combined (cell + location) connected active synapses ≥ segment_activation_threshold, OR
  • Location connected active synapses alone ≥ location_activation_threshold

Setting location_activation_threshold = 8 with max_loc_synapses_per_seg = 12 gives a segment with 8+ mature location synapses the ability to fire from location alone. This is backward compatible — setting the threshold above max_loc_synapses_per_seg disables the feature.

Effect on multi-stride training: Once location alone can make cells predictive, there is no burst-winner competition between strides at shared positions. Each position's correct cells are primed by the location SDR regardless of which stride led there. The O(1000+) epoch requirement for multi-stride convergence should reduce dramatically.

Phase 10, Part B — Learned L5a Displacement Readout

Architecture:

  • L5a (IT neurons) is the conjunctive readout layer. Inputs: L3 active cells (operator context) and L4 active minicolumns (current sensory feature). Neither alone is sufficient.
  • L5a → L5b (ET displacement cells) locally. L5b applies the displacement to L6a. L3 does NOT directly drive L5b.
  • Weight matrix W: (total_L3_cells + num_minicolumns, 1). Maps (L3 ∥ L4) → scalar displacement. Sparse dot product: only active cells/cols contribute.

Learning rule:

  • At step b (when L4 shows the operand number), L5a computes d̂ = W · (L3 active ∥ L4 active). d̂ is applied to L6a.
  • The (L3, L4) state at step b is stored for credit assignment.
  • At the result step, the anomaly score is the teaching signal. Anomaly ≈ 0 → reinforce stored (L3, L4) synapses toward d̂. Anomaly > 0 → weaken them.
  • No backpropagation. The anomaly signal is already computed by the TM and corresponds biologically to prediction error propagated as bursting.

Implementation:

  • New file: l5a_readout.py — L5aReadout class.
  • CorticalColumn.compute() updated to optionally run L5aReadout after L3 TM, before displacement applied to L6a.
  • Training: [a, +, b, =, a+b] sequences, no external displacement injection.

Test targets (test_addition_10.py):

  • A: Reset then jump to position 5 → observe feature(5) → anomaly < 0.1. (validates location-driven prediction)
  • B: Multi-stride training (stride-1 + stride-2 simultaneously) converges in ≤500 epochs. (validates Part A resolves competition)
  • C: Unique-result addition with L5a → anomaly at result < 0.1. (sanity check)
  • D: Ambiguous-result addition (1+9=10 vs 5+5=10) with L5a → anomaly at result < 0.5. (critical — was 1.000 in Phase 9b)
  • E: Generalization to unseen (a,b) pairs → anomaly at result < 0.5.
III

Inter-Layer Wiring — Signal Flow

Wiring
Based on Hawkins local circuits diagram (2024 lecture), Hawkins et al. 2019, and peer-reviewed sublayer connectivity literature (Moberg & Takahashi 2022, barrel cortex studies). Where connections are confirmed by research they are marked; where they remain uncertain they are flagged.

Ascending Sensory Pathway

  • Encoder → L4 (proximal, feedforward, driving): Sensory SDR enters L4. Spatial pooling maps it to a sparse minicolumn representation.
  • L6a → L4 (modulatory, predictive): Grid cell location signal sent to L4. Modulates how L4 responds to sensory input — does not drive L4 firing directly. Combined with orientation, this is how the column predicts what sensory input to expect at the current location. L4 is where feature + location are first combined.
  • L4 → L3 (proximal, feedforward, driving): L4's bound feature-location representation is the feedforward input to L3. This is what drives temporal memory in L3.
  • L4 → L6a (feedback, sensory grounding): L4 projects back down to L6a, enabling sensory anchoring — when a known feature arrives, L6a can correct its grid cell phases to the stored location of that feature.
  • L6a → L3 (basal/distal, modulatory): Location SDR provides basal context to L3 temporal memory cells. Cells become predictive based on the conjunction of prior sequence context AND current location. Implemented via location synapses.

L3 — Sequence Learning and Local Output

  • L3 → L5a (proximal, driving) [confirmed]: L3's active cell SDR drives L5a (IT neurons). This is the confirmed L2/3 → L5a projection in the literature. L3 does NOT directly drive L5b.
  • L3 → neighboring columns' L3 (lateral, driving) [confirmed]: Lateral projections for inter-column voting. Columns whose receptive fields are spatially adjacent vote on object identity here.
  • L3 → L4 (feedback) [confirmed in barrel cortex]: Lower L2/3 projects feedback to L4. Function unclear — may modulate L4's processing of subsequent inputs.
  • L3 → L2 (proximal, driving): L3 feeds upward into L2 (upper L2/3).

L5a — Corticocortical Output

  • L5a → other cortical columns (IT projection) [confirmed]: L5a projects to other cortical columns within the same area and to other cortical areas. Target layers in receiving columns: L2/3 and L5. This is the mechanism for inter-area communication and the hierarchical chain for sensory processing.
  • L5a → striatum (IT projection) [confirmed]: Corticostriatal pathway — the column's output to the basal ganglia for action selection and reward learning.
  • L5a → L2 (local, upward) [confirmed in barrel cortex]: L5a projects back up to L2 (upper L2/3). This is what makes L2 a sensorimotor integration layer — it receives both the sensory sequence (from L3) and the motor context (from L5a).
  • L5a → L6b (per diagram) [uncertain]: The Hawkins diagram shows L6b receiving from L5a. The computation performed by L6b with this signal is not understood.

L5b — Displacement Module Output

  • L5b ↔ L6a (bilateral) [confirmed, Hawkins 2019]: Each L5b displacement cell module pairs with its corresponding L6a grid cell module. Firing displacement d shifts each L6a module's phase by (d mod λᵢ) — path integration. This is the bidirectional link between displacement and reference frame.
  • L5b → subcortical motor targets [confirmed]: Thick-tufted ET neurons project to brainstem, spinal cord, superior colliculus. Direct muscle control pathway.
  • L5b → thalamic relay cells → next cortical area L4 [confirmed]: The second axon branch relays the displacement signal to the next hierarchical cortical area via thalamus. This is how the sensorimotor loop propagates up the hierarchy — both the physical movement and the signal about that movement reach the next level simultaneously.
  • L5b → L4 (with orientation) [proposed, Hawkins 2019]: Combined with an orientation signal, L5b displacement modules project to L4 to predict the next expected sensory input.

L2 — Long-Range Output

  • L2 → other cortical areas (long-range) [confirmed]: Upper L2/3 projects to other cortical areas (targeting L2/3 of receiving areas). L2 carries the integrated sensorimotor representation — sensory sequence (from L3) combined with motor context (from L5a) — to distant regions.

L6 — Thalamic Pathways

  • L6a → thalamus (modulatory feedback) [confirmed]: L6a projects back to the thalamic nucleus providing sensory input to this column. Modulatory only — modifies how relay cells respond to subcortical input, does NOT cause them to fire directly.
  • L6b → thalamus (routing) [confirmed]: L6b projects to the same thalamic nucleus as L6a, plus forward to the next higher-order thalamic nucleus. Function unclear — possibly routing for hierarchical level-skipping.
  • Thalamus → L4 (sensory relay) [confirmed]: Thalamic relay cells deliver sensory input to L4. The thalamus is a relay with modulation, not a passive pipe — its relay properties are shaped by L6a feedback.

External Connections

  • Old Brain → Encoder → L4: Sensory data flows from the Old Brain through encoders before reaching L4.
  • L5b → Old Brain: Motor command exits to Old Brain for decoding and muscle activation.
  • Higher cortical regions → L1 → apical dendrites: Top-down feedback via L1. Not implemented in initial build.

Connection Type Key

  • Driving (proximal): Causes cells to fire. L4 → L3, L3 → L5a, L5b → subcortical.
  • Modulatory (basal/distal): Affects how cells respond to driving input but does not cause them to fire alone. L6a → L4, L6a → L3, L6 → thalamus.
  • Lateral (same level): Between cells in the same layer across columns. L3 ↔ L3 (voting).
  • Feedback: From output back toward input. L3 → L4, L5a → L2.
