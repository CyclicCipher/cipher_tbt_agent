The Learning Algorithm
A Merge Plan

Ingredients: Items 52, 85, 92 Priority: merge 52 + 92; fold in 85 if possible Constraint: 4GB VRAM · continual learning needed

Three candidate directions for the training algorithm sit far apart, because they operate on different objects and two of them abolish the very thing the third depends on. This document states the tension exactly, then plans a route to merge the priority pair — item 52 and prospective configuration — with DiffusionBlocks as a stretch wrapper. The organizing idea is that all three are denoising operations on the learning process; the work is making their denoisings compose.

00

The Goal

The training algorithm has to do three things at once, and no standard recipe does all three: generalize well from little data (the budget is small, the knowledge base is small), resist catastrophic forgetting (a world model learned through experience is inherently online and incremental), and fit in 4GB. Each candidate ingredient attacks one face of this:

Item 52 — generalization
Suppresses memorization, accelerates grokking 5×, estimates generalization without a validation set. The anti-memorization face.
Prospective configuration (92) — forgetting
Settle-then-update credit assignment that resists catastrophic forgetting and suits online / changing-environment learning. The continual-learning face.
DiffusionBlocks (85) — memory
Block-wise training decoupling memory from depth. The 4GB face — and a natural fit for recurrent depth.

If all three can be merged, do it. If not, the non-negotiable pair is item 52 + prospective configuration — generalization and continual learning are the two faces we cannot ship without. The unifying lens that makes the merge conceivable: all three are denoising/filtering operations on the learning process, each on a different object — representation level (85), spectral SNR (52), energy-consistency (92). The task is to make the filters compose rather than collide.

01

The Three, Honestly

1a   Item 52 — A Theory of Generalization established

The empirical NTK partitions the output space into a signal channel (error dissipates fast) and a noise reservoir (near-zero eigenvalues trap residual error test-invisibly). Coherent population signal accumulates across the minibatch; idiosyncratic memorization is suppressed into a slow random walk. In practice this reduces to an SNR preconditioner on top of Adam — one extra state vector, near-zero cost — that suppresses the noise channel, accelerates grokking 5×, and yields a validation-free population-risk estimate from a single run. What it operates on: the backprop gradient. The signal/noise partition is an NTK construction, and the NTK is built from the Jacobian — a gradient object. This is the fact that makes it hard to combine with the other two.

item 52 (arXiv:2605.01172)representation: essential/accidental

1b   DiffusionBlocks — Block-Local Training directional

Splits the network into blocks trained independently by treating the forward pass as a diffusion denoising trajectory: each block is responsible for one denoising step / noise level, which gives it a well-defined local target without end-to-end backprop. Memory is decoupled from depth — you hold one block's activations at a time. What it operates on: block-local targets derived from the diffusion schedule. It abolishes the global backward pass in favor of per-block objectives.

item 85 (Sakana)architecture §4a

1c   Prospective Configuration — Settle, Then Consolidate directional

An energy-based / predictive-coding learning rule. Clamp the output to the target, let the network settle — relax neural activities to the equilibrium that minimizes energy and is consistent with the target — and only then update weights, locally, to consolidate the settled activities. Activity changes before plasticity; weights move toward a configuration the network has already found consistent, so updates interfere less with prior knowledge — hence resistance to catastrophic forgetting. What it operates on: local prediction errors at a settled equilibrium. There is no global backward pass; credit assignment is local, and the settling is a relaxation to convergence — the same fixed-point computation as depth recurrence / DEQ.

02

Why They're Distant

The distance is not stylistic; it is structural. Item 52 preconditions a global backprop gradient. Prospective configuration and DiffusionBlocks both abolish the global backprop gradient. Item 52's preconditioner needs the object — the gradient, and the NTK built from it — that the other two are specifically designed to do without. You cannot stack 52's preconditioner on prospective configuration, because there is no global gradient there to reweight. They live in different worlds: 52 in the backprop/NTK world, 92 and 85 in the local-learning world.

Item 52's implementation (a preconditioner on Adam) is welded to backprop. But its contribution is a theory — a claim about which directions carry generalizing signal and which carry memorization noise. A theory is portable in a way an implementation is not. The merge does not stack 52's code onto 92; it re-instantiates 52's signal/noise decomposition inside 92's local-learning world, applied to the local settled updates instead of to a global gradient. Whether that re-instantiation exists is the central open question (§3c, L1).

The reason to believe the door is real: all three are, at heart, the same kind of operation seen from different angles — separating what to keep from what to discard during learning. DiffusionBlocks separates clean representation from noise across depth (denoising per level). Item 52 separates signal from noise across the spectrum (NTK eigendirections). Prospective configuration separates consistent from interfering change across energy (settling to a low-energy configuration discards interference). Three denoisings on three objects. If they can be made to act on a shared pipeline, they compose into a single layered denoising of the learning process rather than three rival algorithms.

The Priority Merge: Item 52 + Prospective Configuration

3a   Merge the Theory, Not the Implementation plan

Prospective configuration supplies the credit-assignment mechanism: settle to a target-consistent equilibrium, then update weights locally, resisting catastrophic forgetting. Item 52 supplies the per-update generalization filter: suppress the components of the update that carry memorization noise, preserve those that carry coherent signal. The merge applies item 52's signal/noise decomposition to prospective configuration's local settled updates rather than to a backprop gradient. PC says which direction to move (the settled, interference-free direction); item 52 says how much of that move is signal vs. memorization and scales accordingly.

3b   Two Complementary Noise Filters directional

The reason this is worth the trouble is that the two attack different failure modes with different filters, and the failure modes are linked:

Item 52 filters memorization

  • Per-example idiosyncratic update components that don't generalize (the noise channel).
  • Filter axis: spectral / statistical SNR across the minibatch.

PC filters interference

  • Update components that overwrite prior knowledge (catastrophic forgetting).
  • Filter axis: energy-consistency of the settled configuration.

These are synergistic, not merely compatible. Memorized content is exactly the brittle, idiosyncratic content most likely to be overwritten later — so suppressing memorization (52) reduces the amount of fragile knowledge that can be forgotten, which strengthens PC's anti-forgetting (92) rather than just running beside it. Composing the two filters yields a learning rule that suppresses memorization and interference at once.

3c   The Theoretical Gap (The Central Risk) open — falsifier

Item 52's signal/noise partition is an NTK construction, computed from the Jacobian of outputs with respect to parameters — a backprop object. Prospective configuration computes no such Jacobian; its updates are local products at a settled equilibrium. So the merge requires an analogue of the NTK signal/noise decomposition for energy-based local learning: a way to relate the across-minibatch statistics of PC's local settled errors to the generalizing-signal-vs-memorization-noise partition. This is genuine theoretical work, and it might not exist — the NTK analysis may not transfer cleanly to the settled-equilibrium dynamics.

If the settled-equilibrium updates do not admit a signal/noise decomposition analogous to the NTK's — i.e. if "coherent across the minibatch" in PC's local errors does not track "generalizing" the way item 52 proves it does for gradients — then there is nothing for item 52's idea to act on, and the merge fails at the first step. This is the one assumption the whole plan rests on. It should be tested before anything is built on top of it.

3d   The De-Risking Path: Measure Before Intervene plan

Item 52 yields two things from its decomposition: a measurement (the validation-free population-risk / generalization estimate) and an intervention (the SNR preconditioner). The measurement is the safer first target, and it directly tests §3c:

1
Port the decomposition as a monitor. Implement item 52's signal/noise estimate over prospective configuration's local updates and use it only to measure generalization during PC training. If the estimate tracks real held-out generalization, the decomposition transfers — L1 is answered affirmatively at low cost, with nothing yet riding on it.
2
Turn the monitor into a preconditioner. Only once the measurement is validated, use the same signal/noise estimate to scale PC's local updates — suppress the noise-channel components. This is the actual merge.
3
Validate on continual learning. Test the synergy claim (§3b): does memorization-suppression measurably strengthen PC's resistance to catastrophic forgetting on a sequential-task benchmark, versus PC alone?

04

The Stretch: Folding In DiffusionBlocks

If the priority pair works, DiffusionBlocks can in principle wrap it as a third filter that also solves the memory face. The layering, outermost to innermost:

1
DiffusionBlocks sets the block targets and decouples memory. The diffusion interpretation gives each block a denoising target — effectively the clamp for that block — and lets the system train one block at a time, which is what makes the whole stack fit in 4GB.
2
Prospective configuration settles each block to its target. Within a block, settle to the equilibrium consistent with the diffusion-defined target, then local-update. PC supplies the credit assignment; the block target supplies the clamp.
3
Item 52 preconditions the local updates. The signal/noise filter scales the per-block local deltas, suppressing memorization, exactly as in §3.

This is the layered denoising from §2 realized as one pipeline: DiffusionBlocks denoises across representation level, prospective configuration denoises across energy-consistency, item 52 denoises across spectrum. Three orthogonal denoisings on the same forward/settle/update pass. Conceptually they belong together; the difficulty is purely in the mechanics of making one block's settle-and-filter respect another block's diffusion target.

DiffusionBlocks wants block independence; prospective configuration's settling is global relaxation — message-passing across the whole network to reach a consistent equilibrium. These pull against each other. The candidate reconciliation is block-local settling: each block settles only to its own diffusion target, independently, sacrificing global consistency for memory and modularity. Whether block-local settling retains PC's anti-forgetting — which may depend on the global consistency it normally enforces — is the open question that decides whether all three can coexist (L4). It is entirely possible that 52 + 92 merge cleanly while 85 must stay a separate memory technique applied around them rather than fused in.

Architectural Unification

The merge is not just compatible with the architecture in the companion document — it is the same machinery seen from the learning side.

Settling = depth recurrence
Prospective configuration's relaxation-to-convergence is the same fixed-point computation as the model's Ouro/DEQ depth recurrence. The architecture's own recurrence can be the settling engine — the concrete answer to item 91 (efficient settling). Inference (clamp input, settle) and learning (clamp target, settle) become one relaxation with different clamps.
SNR = essential vs. accidental
Item 52's signal/noise partition is the optimizer-level form of the differentia/accident distinction from the representation document. What relational attention enforces architecturally, item 52 enforces in the learning dynamics. Same principle, two levels.
JEPA = the clamp target
Prospective configuration needs a target to clamp. In the world model, that target is the predicted (or actual) future latent. So JEPA supplies the clamp — which makes item 90 (fuse DiffusionBlocks + JEPA into a prospective-configuration algorithm) precisely the subject of this document.
One energy, three descents
Active inference's frame (objectives §4) predicts the whole system should be descent on a single energy. Settling, planning, and learning would all be relaxations on that energy — the unification is the falsifiable commitment, not a metaphor.

The strongest version of the claim: in this design, inference and learning are the same relaxation process. The depth recurrence settles to an attractor given the input (inference); the same recurrence, with the target clamped, settles to a target-consistent configuration whose consolidation is the weight update (learning). Item 52 filters which part of that consolidation is signal. If this holds, the model does not have a separate "training mode" and "inference mode" — it has one settling operation used two ways, which is exactly what an energy-based, active-inference system should look like.

Open Questions & Falsifiers

L1 Does item 52's signal/noise decomposition have an analogue for energy-based local learning? open — central falsifier

The NTK partition is a gradient construction; prospective configuration has no global gradient. The entire merge depends on whether "coherent across the minibatch" in PC's local settled errors tracks "generalizing" the way item 52 proves it does for backprop gradients. Test it as a measurement first (§3d step 1). If it fails, 52 and 92 cannot merge and must be pursued separately.

item 52item 92
L2 Can the model's depth recurrence serve as an efficient settling engine? directional

If settling reuses the Ouro/DEQ recurrence, prospective configuration becomes nearly free architecturally (item 91). But this requires the recurrence to converge reliably and fast enough that per-update settling is affordable. If settling is too slow, PC is impractical at any useful scale regardless of the item-52 merge.

item 91item 1 (DEQ)
L3 Does memorization-suppression actually compound with PC's anti-forgetting? open

The synergy claim (§3b) is that suppressing memorization reduces the brittle content available to be forgotten, strengthening PC. It is plausible but unproven; the two could instead be redundant (both already removing the same updates) or even antagonistic (52 suppressing updates PC needs to settle). Decided empirically on a sequential-task benchmark (§3d step 3).

item 52item 92item 79
L4 Can block-local settling retain anti-forgetting, allowing all three to fuse? open

DiffusionBlocks wants block independence; PC's settling is global. Block-local settling buys memory and modularity but sacrifices the global consistency PC may rely on for its anti-forgetting. If anti-forgetting survives block-local settling, all three fuse (item 90); if not, DiffusionBlocks stays a separate memory wrapper applied around a 52+92 core rather than fused into it.

item 85item 90item 92
L5 Is inference-equals-learning (one settling, two clamps) actually realizable? theoretical

The strongest unification (§5) claims the depth recurrence is both the inference settling and the learning settling, differing only in what is clamped. This is elegant and matches the active-inference frame, but whether a single relaxation operator can serve both roles without destabilizing either — and descend a single shared energy — is unproven. It is the through-line that would make the architecture and the learning algorithm one object rather than two.
