Reservoir Computing Integration
Proposals & Experiments — A Research Fork

Problem: the time-series / streaming-temporal gap (item 98) Status: fork — not on the v1 critical path Preferred: P3 (hybrid mixer), pending neuroscience

The architecture has thinking-time recurrence (depth, chain) but no native streaming-temporal substrate, and attention is the wrong tool for it. Reservoir computing is the candidate fix, but how a fixed dynamical reservoir slots into a gradient-trained reasoning model is genuinely unresolved — it might be a front-end, a parallel axis, an interleaved mixer, or a structural prior. This document lays out the integration problem honestly, gives four proposals with proposed experiments, and parks the deeper question — what the brain does — as the path to a better answer, to be taken up another day.

00

The Fork

This is a research branch, deliberately off the v1 critical path. The v1 blueprint (architecture §6) stays reasoning- and text-first; reservoir computing is spun up when time-series and streaming dynamics come into scope. The reason it is a fork and not a feature is that it is hard in a way the rest of the architecture is not: a reservoir is a different kind of object from a gradient-trained transformer, and the right way to join them is unknown.

The direction — reservoir computing for the temporal substrate — is the established part: echo-state networks and liquid-state machines are decades-proven on temporal and chaotic-signal tasks, training only a readout, BPTT-free. HypercubeESN (item 98) is a memory-frugal, reproducible, edge-friendly candidate implementation of that direction (XOR-addressed hypercube wiring, a per-neuron delay line, the whole reservoir specified by a handful of scalars), not a settled dependency. What is open is the integration: how a frozen reservoir couples to a gradient-trained, convergent reasoning core. That is this document's subject.

01

Two Hard Problems Every Proposal Must Answer

1a   The spectral-regime incompatibility theoretical

A reservoir and the DEQ/attractor core sit at opposite ends of the spectral-radius axis, and this is the deepest fact in the whole problem. The echo-state property requires spectral radius ≈ 1 — edge-of-chaos dynamics that do not die, so temporal history stays alive in the state. The DEQ core (architecture §6a) requires contraction, spectral radius < 1, so it converges to a fixed point that is the answer. One dynamical system cannot occupy both regimes. The consequence is not an engineering inconvenience but a structural verdict: the reservoir must be a distinct component with its own dynamics, feeding or running beside the convergent core — never a re-tuning of it. This rules out replacing the core's dynamics with reservoir dynamics, and it is why the viable proposals all keep two separate dynamical regimes.

1b   The frozen-to-trained seam open

A reservoir is frozen; it trains only a readout — that is the entire source of its efficiency (cheap, BPTT-free, reproducible). The rest of the model is gradient-trained end to end. So every integration is really a decision about where the frozen-to-trained seam goes and how gradients route around the frozen reservoir. Put the seam at the input (P1), at a side-memory interface (P2), inside the residual stream (P3), or dissolve it by making the reservoir trainable (P4) — each choice trades integration cleanliness against how much of the reservoir's cheapness survives. There is no free option; the proposals differ in which cost they pay.

02

Integration Proposals

Four proposals, ordered by invasiveness. Each names where the seam goes, what it costs, and a concrete experiment. The full-replacement option — a reservoir instead of attention everywhere — is excluded by §1a and by capability: reservoirs have recency-biased fading memory and no content-based addressing, so they cannot do the relational binding attention provides, and the project will not give up composition.

2a   P1 — Frozen Temporal Front-End directional

Setup. The streaming signal drives a frozen reservoir; a thin trainable readout projects the reservoir state into the shared latent space as tokens; the attention/DEQ core processes them as usual. This is architecture §6b with the time-series modality's adapter being a reservoir instead of a linear projection — the classic ESN forecasting setup. Seam: at the readout; gradients stop there, reservoir frozen. Cleanest. Cost: temporal processing is pre-core and static — the model reasons over a fixed temporal summary it cannot dynamically interrogate, and the fading-memory profile is fixed, not learned.

Frozen HypercubeESN reservoir → trainable linear readout → tokens → existing core. Compare three conditions on a forecasting / nonlinear-temporal task (NARMA-N, Jaeger memory-capacity, a real forecasting benchmark): core-with-reservoir-features, core-alone, and reservoir-alone (ESN linear readout). The decisive question is narrow and cheap: do frozen reservoir features give the core a temporal capability it lacks? If even as a front-end they do not help, the whole direction is suspect; if they do, escalate.

2b   P2 — Parallel Queryable Axis (the genuine 3rd axis) directional

Setup. A frozen reservoir runs alongside the chain, updated on the world/token clock (not the depth-iteration clock — depth is thinking-time, the reservoir is world-time), maintaining persistent temporal state the core reads via cross-attention. The model then has depth recurrence (settle), chain recurrence (step), and a reservoir state (temporal memory) it can attend into throughout reasoning. Seam: the trainable cross-attention/readout into the core; reservoir frozen. Cost: more machinery (cross-attention to a non-token state, clock management), and the frozen features may be misaligned to what the core wants.

Add the world-clock reservoir + cross-attention to a core that passed P1. Test on a task needing both relational reasoning and temporal recall — a streaming environment from the training-environment doc, or a synthetic task interleaving relational inference with temporal-order recall. Measure whether queryable temporal memory extends horizon (architecture §1g) and beats P1 on long-range temporal dependencies, and watch the clock separation carefully.

P2's "queryable temporal memory the core attends into" overlaps with item 79's hippocampus/CLS memory, which is also a queryable store the core reads. These may be one component wearing two hats (temporal-reservoir and episodic-hippocampal memory) or two distinct systems. Decide before building, to avoid two parallel memory systems that should have been one.

2c   P3 — Interleaved Hybrid Mixer open — preferred

Setup. Make temporal mixing and relational mixing first-class peers in the same stack: attention/relational layers do content/relational mixing (composition — what reasoning needs), reservoir layers do temporal mixing (sequential dynamics), interleaved. This is the most principled option — temporal context is available at every layer, shaping relational computation directly, rather than being relegated to a front-end (P1) or a side-memory (P2). Seam: inside the residual stream — a frozen, non-differentiable layer wedged into a gradient-trained stack, which is where the friction is sharpest. Cost: the interface (how reservoir output enters the residual stream, how gradients route around it) and the risk that the reservoir's fixed dynamics do not match the representation at that depth.

Two reasons it is the preferred direction despite the seam problem. First, it is the known-good pattern with one component swapped: hybrid recurrent-attention stacks (interleaving SSM/Mamba-style recurrent mixers with attention) already work well. A reservoir is the fixed, nonlinear extreme of that recurrent mixer — so P3 is "hybrid recurrent-attention" with the recurrent mixer being a reservoir, not an exotic new structure. Second, it is the most biologically resonant: cortical microcircuits combine recurrent (reservoir-like) and feedforward processing in the same tissue. If the brain interleaves, P3 is the faithful model — which is exactly why the better answer needs the neuroscience (§4).

Exploit the known-good pattern as a smooth path. Start from a working hybrid stack with a trained recurrent mixer (SSM/Mamba-style) interleaved with relational attention; confirm it works. Then progressively freeze it toward a reservoir: reduce trained parameters, fix the spectral radius near 1, adopt the hypercube structure — measuring at each step where temporal capability holds and where it breaks. The decisive question: can a frozen reservoir mixer match a trained recurrent mixer's temporal contribution while keeping the reservoir's cheapness? If performance collapses as you freeze, P3-pure is not viable and the fallback is P4 (this experiment also dissolves continuously into P4).

2d   P4 — Reservoir-as-Initialization (the bridge) directional

Setup. Use HypercubeESN's structure — hypercube wiring, XOR addressing, spectral-radius-scaled init, delay-line memory — as the initialization and architectural prior of a normal differentiable recurrent layer, then train it. Seam: dissolved — it is just a layer with a good temporal prior, fully differentiable, integrates trivially. Cost: you forfeit the reservoir's defining advantage — train-only-the-readout, BPTT-free, ultra-cheap, reproducible — and inherit backprop-through-time's vanishing-gradient problems the reservoir avoided. A different trade: differentiability and integration for cheapness and reproducibility. The memory-frugal hypercube structure can still be kept as the prior.

Initialize a trainable recurrent layer with HypercubeESN structure + spectral scaling; train end-to-end. Compare against a random-init recurrent layer and against the frozen reservoir (P1/P3). Measure whether the reservoir prior beats random init, and how much temporal capability survives training (does the spectral radius drift away from the good regime once gradients touch it?).

03

Experimental Program

The proposals form a natural escalation, cheapest and most decisive first. The program is gated: each stage only runs if the prior one showed value.

1
P1 — does the core gain a temporal capability at all? The highest-information, lowest-cost probe. If frozen reservoir features do not help the core even as a front-end, stop — the direction is wrong. If they do, the rest is worth pursuing.
2
P2 — does it need to be live during reasoning? Only if P1 showed value but the core wants temporal memory it can interrogate mid-reasoning, not just a static summary. Resolve the item-79 overlap first.
3
P3 via the SSM bridge — can a frozen mixer match a trained one? The preferred direction, approached by freezing a working trained-recurrent hybrid toward a reservoir. Continuously dissolves into P4 as freezing fails, so the two share one experimental track.
4
P4 — the fallback. If the frozen seam cannot be made to work in-stack, trade cheapness for a trainable reservoir-initialized layer.

Three standard reservoir benchmarks isolate the temporal contribution cleanly, and HypercubeESN ships them: Jaeger memory capacity (how many past steps the state retains), NARMA-N (nonlinear temporal modeling), and streaming forecasting / anomaly detection. The one knob to monitor throughout is the spectral radius — it is the single parameter that decides whether the component is in the memory-preserving regime (≈ 1) or has slipped toward forgetting (< 1) or chaos (> 1). Every proposal lives or dies by keeping the reservoir component in the right regime while the core stays contractive.

04

The Deeper Questions — Deferred to Neuroscience

The proposals above are how to try things; they do not tell us which is right. The better answer requires understanding how the brain integrates reservoir-like temporal dynamics with structured computation — because biology has already solved this, and the single most important architectural choice (P3 interleave vs P2 separate subsystem) is exactly the choice the brain's organization would settle. This section is deliberately parked, to be taken up on a later day with proper neuroscience research.

The discriminating question, stated plainly: does the brain interleave temporal and relational processing in one circuit (→ P3), or separate them into distinct subsystems (→ P2)? The known anatomy points both ways, which is precisely why it needs real study rather than a guess:

Evidence pointing toward P2 (separate subsystem)

  • The cerebellum as a liquid-state machine. A random granule-cell expansion feeding a trained Purkinje readout is a biological reservoir-with-trained-readout, coupled to but separate from cortex — a concrete template for the frozen-reservoir + readout + reasoner coupling.
  • Cerebellum as forward model. If the cerebellum is the brain's temporal/predictive subsystem feeding the cortical reasoner, that argues a separate reservoir subsystem, not interleaving.

Evidence pointing toward P3 (interleaved in one circuit)

  • Cortical microcircuits as reservoirs (Maass LSM). If cortex itself is partly reservoir-like, temporal and relational mixing already coexist in one tissue — the faithful model is then interleaving.
  • Recurrent + feedforward in the same column. Cortical tissue combines recurrent dynamics and feedforward selectivity locally, which is exactly the P3 hybrid-mixer picture.

Supporting questions worth the same deferred study:

i
Thalamo-cortical loops. How does the thalamus gate and route temporal/contextual signals into cortical computation? A model for the reservoir→core interface — whether it is cross-attention (P2) or in-stream injection (P3).
ii
Time cells, sequence replay, theta phase coding. How is temporal order represented and made queryable in hippocampus and entorhinal cortex? Informs P2's queryable memory and the item-79 overlap — and connects to grid-cells-as-Yoneda from the representation document.
iii
Timescale separation and neuromodulation. How does the brain run fast reservoir-like dynamics and slow consolidation at once? Directly the world-time-vs-thinking-time clock problem (P2).

Open Questions

R1 Do frozen reservoir features give the core a temporal capability it lacks? open — decisive

The gate on the entire fork (P1 experiment). If a frozen reservoir front-end does not measurably extend what the core can do on temporal tasks, no more elaborate integration will rescue it. Cheap to answer, and it should be answered first.
