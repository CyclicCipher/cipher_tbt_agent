00

The Core Commitment: Recurrent Depth

There is no variant of this model that is not a recurrent-depth model of some kind. This is the one fixed point of the design. Everything else — the attention mechanism, the activation, the training objective — is negotiable; recurrent depth is not.

The justification is threefold. First, the bitter lesson: a fixed-depth network has a fixed compute budget per token regardless of problem difficulty, which forces it to pattern-match rather than compute. Recurrence lets the model spend more computation on harder inputs — emergent search rather than hand-coded depth. Second, the companion document's central claim is that reasoning is the composition of relation-transformations (morphisms); composition is inherently iterative, so the architecture that realizes it must iterate. Third, the memory constraint: weight-shared recurrence buys effective depth without parameter growth, which on 4GB VRAM is not a luxury but the only way to reach the depth the problem needs.

The critical distinction the field tends to blur: there are two different things "recurrence" can mean, and they are orthogonal. Depth recurrence (Ouro/LoopedLM) loops the same block to deepen the computation of a single step — "think harder about this." Thought recurrence (COCONUT) feeds a latent output back as the next input to extend a reasoning chain — "take another step." They are not redundant and they are not the same axis. The plan uses at least one; ideally both, nested.

Depth axis — "think harder"

Same weights applied L times to refine a representation in place. Fixed input, deepening transformation. A fixed-point / equilibrium computation. Ouro, LoopedLM, Universal Transformer (item 1).

Thought axis — "take a step"

Latent output fed back as next input, extending a trajectory of continuous thoughts that attend to each other. Chain length = reasoning length. COCONUT (item 2).

The Recurrence Stack

1a   Depth Recurrence — Ouro / LoopedLM established

A block B with shared weights applied repeatedly: hl+1 = B(hl) for l = 0…L−1 on a fixed input. Effective depth L at the parameter cost of one block — Ouro reports roughly 4× parameter efficiency versus an unrolled network of equivalent depth. The natural mathematical reading is a fixed-point computation: the loop iterates toward an equilibrium h* = B(h*), which is the Deep Equilibrium (DEQ) view and matters enormously for memory (§4b).

This is also the substrate for the attractor-state picture in the companion document: KG-style objects are attractors of the depth loop, and a relation is the transformation moving the state from one attractor toward another (item 41).

item 1item 41 (attractor models)

1b   Thought Recurrence — COCONUT directional

Instead of decoding a hidden state to a token and re-embedding it, COCONUT feeds the last hidden state back directly as the next input embedding — reasoning in continuous latent space. The sequence grows as [context, c1, c2, …, ct]; only after the latent chain completes does the model decode an answer. The payoff is that a continuous thought is not forced through the bottleneck of a discrete token vocabulary, so a single thought can carry more than one token's worth of information and need not commit prematurely to one branch.

The companion document sharpens what these thoughts should be (§2 below): not compressed content, but relation-transformations — operators.

item 2item 55 (abstract CoT)

1c   Can They Combine? open — central

The question is whether depth recurrence and thought recurrence co-exist in one model. The argument that they can: they are orthogonal axes, so the natural combination is nested — each COCONUT thought step ct is itself computed by an Ouro-style depth loop. Total computation scales as T (thoughts) × L (depth loops per thought). Because depth refines a step and thought extends the chain, the two are complementary, not duplicative; a capable reasoner needs both deep per-step computation and multi-step chains.

The honest hard parts, in order of severity:

1
Gradient flow and memory. Backprop through a T × L unroll is the central obstacle on 4GB. Two escapes, both real: DEQ-style implicit gradients give O(1) memory in the depth loop (§4b), and DiffusionBlocks decouples memory from unroll depth (§4a). Without at least one, the nested model does not fit.
2
Stability. Nested recurrence can blow up or collapse over many iterations. Requires bounded/normalized signal propagation — RMSNorm, QK-Norm, and possibly a bounded periodic activation (§3b, §3c) — plus the anti-collapse term from the JEPA objective.
3
Halting. When does the depth loop stop (per thought), and when does the thought loop stop (per problem)? This is what HRM's ACT halting is for, applied at both recurrence axes (§1e) — adaptive computation time per step and per problem.

It is genuinely unknown whether the two co-train cleanly. The orthogonality argument says they should be combinable in principle; it does not guarantee gradient descent finds a joint solution. The fallback is graceful: if nesting fails, ship depth recurrence alone (the more established of the two) and treat thought recurrence as a later addition. At least one axis is present; both is the stretch goal.

1d   JEPA as the Objective — A World Model in Latent Depth directional / ideal

JEPA is not a module to bolt on; it is the training objective that shapes the latent thoughts. Predict the representation of a future state from the current one, in latent space, rather than reconstructing tokens or pixels. In a recurrent-depth reasoner this means: given the current latent state, predict the latent state that a reasoning step would produce. The model learns to run its own world-model forward in its head — imagine consequences without committing to tokens. This is precisely the "telos" placement from the companion document: purpose lives in the objective, not in a head.

This connects three agenda items into one objective: active inference (item 20, represent things by their role in achieving goals), MuZero-style latent planning (item 77, "an HRM with a JEPA objective gets ~80% of the way there"), and Sub-JEPA's anti-collapse machinery (item 49). The anti-collapse term is mandatory: a model predicting its own future latents has a trivial collapse solution (constant latent), defeated by SIGReg/VICReg-style isotropy regularization.

Whether a JEPA predictive objective can drive a COCONUT latent chain — and do so through a depth-recurrent core — without either collapsing or destabilizing is the least-tested part of the whole plan. COCONUT was originally trained with a token-replacement curriculum, not a predictive latent objective. Making JEPA the driver is the ideal; it is not known to work. Flagged honestly as the highest-risk, highest-reward element.

1e   HRM — Take the Outer Loop, Make the Hierarchy Optional directional

The earlier framing leaned on HRM's slow/fast hierarchy as the controller. The ablation evidence has redrawn what is worth taking. The ARC Prize analysis and two mechanistic follow-ups found the H/L hierarchy had minimal impact versus a similarly-sized plain transformer, while the under-documented outer refinement loop — repeated forward segments with deep supervision and ACT halting — drove the gains: iteration is the engine, hierarchy is secondary. HRM-Text (1.15B, ~40B tokens, runs at 0.6 GiB int4) carries the same recurrent depth to language but, tellingly, has not had the flat-vs-hierarchical control run on it either.

So we take from HRM the parts the evidence supports — and which, notably, could enhance any architecture: the outer refinement loop, deep supervision over segments, ACT halting, and the one-step / equilibrium gradient (already §1a, §4b). These are the recurrent-depth machinery, not the module split. The H/L hierarchy itself becomes an optional module — built last and tested against single-timescale recurrent depth (A7). The default build is single-timescale, consistent with the objectives staging.

What survives independent of the hierarchy is the control role: a halting / policy layer deciding how hard to think per step and how many steps to take. That is where metacognition lives (item 80) and is the Kantian apperceptive binding from the companion document — the "I" that owns and binds the latent trajectory. It requires a control policy over the recurrence (§1f), not specifically the slow/fast split.

1f   AutoTTS as the Control Policy — and Halting Without an EOS Token directional

AutoTTS (item 8) is "the bitter lesson applied to chain-of-thought — when to terminate or branch, and how to use branches." That is not a separate module to add; it is precisely the policy the halting/control layer (§1e) should learn over the two recurrence axes — a control policy that does not depend on the H/L hierarchy. Getting its effect means making three decisions learned and adaptive rather than fixed: depth-loop count per step, thought-chain length, and when to spawn and prune divergent branches. The branch-evaluation half pairs naturally with item 82 (ADHD parallel/divergent thinking), whose value is spotting reasoning traps — i.e. deciding which divergent branches to kill.

Halting without an EOS token. A model that thinks (and ideally speaks) in latent space has no natural discrete token to mark "stop." Replace the EOS token with a continuous halting signal. Two mechanisms: an ACT-style accumulated halting probability the controller emits per step, or — more natural for a JEPA model — latent fixed-point convergence: stop when the predicted next latent ≈ the current one (nothing left to predict) or when the depth loop reaches equilibrium. Under the active-inference frame this is principled: stopping = free energy minimized = the goal state reached. The EOS token is a crude proxy for "goal achieved"; convergence-based halting makes "done" mean "predicted future = preferred future," and reuses the same halting machinery as the recurrence.

1g   Horizon — Hierarchical JEPA + Test-Time Scaling open hunch

LeCun argues hierarchical JEPA is required for long-timescale prediction, and his experiments support it — but they increase the achievable planning length without extending it indefinitely, because the number of hierarchy levels is fixed at training, so horizon is architecturally bounded. The hunch worth testing: test-time-scaling techniques (COCONUT, AutoTTS, and modifications) could extend not just reasoning depth but the length of the predictions and actions the model can make.

The hunch is true in direction, with one honest limit. Test-time recurrence is dynamic where hierarchy is static: a recurrent JEPA can roll its world model forward at inference (predict next latent → feed back → predict again), extending horizon without retraining. So TTS does push past fixed hierarchical JEPA. But the binding limit then becomes error accumulation in the latent rollout, not architecture — the same compounding that limits autoregressive generation. The two combine better than either alone: hierarchy reduces the number of error-accumulating steps per unit horizon (coarser levels = fewer steps), and TTS spends variable compute to roll further when needed. Not indefinite, but further and adaptive.

Two complementary mechanisms could bound the compounding error. (1) Operator structure: composing clean relation-transformations (§2) should accumulate less error than predicting noisy content vectors — if so, the relational design is itself a horizon-extension mechanism, not only a reasoning one. (2) Targeted TBAF: the triangle activation's stated purpose is precisely to correct autoregressive drift, so a single strategic placement on the rollout feedback path (§3b) could cap per-step error growth at the exact point where drift enters. Whether either transfers to this latent space is the most interesting untested claim in the recurrence stack.

Latent Thoughts as Operators

What are the continuous thoughts in the latent loop? The default assumption — that a thought is a compressed content vector, a squeezed token embedding — is probably wrong, and two agenda items point the other way.

Abstract Chain-of-Thought (item 55) reports ~12× token savings by reasoning over abstract tokens that are explicitly not equivalent to language tokens. Because the method is non-interpretable we cannot read off what those tokens are — but the efficiency ratio is a clue too large to ignore. A 12× compression over natural language is exactly what you would expect if each abstract token were an operator rather than a piece of content: the natural-language description of "compose relation R then relation S" is many tokens, but the operation itself is one step. Content compresses linearly; operations compress combinatorially.

If reasoning is composition of relation-transformations (companion §3), then a latent thought should be parameterized as a morphism — a relation-transformation applied to the current state, ct = Mrt · state — not as compressed content. This unifies COCONUT with the relational attention: the thought is the morphism. It explains item 55's two observations at once — why the abstract tokens are not language tokens (they are operators, not objects) and why they are ~12× more efficient (an operation is more compact than its description). The KG algebra of item 34 is the space these operators live in.

This also answers Chollet (item 86) on his own terms. His claim that connectionist models cannot do symbolic descent is too strong: transformers have been trained to simulate C execution at arbitrary depth, so a recurrent-depth model whose latent steps are operator-applications is literally executing a program in latent space. Symbolic descent is not foreclosed to connectionist models; it is what a relation-composing recurrent loop does. The data-efficiency gap Chollet identifies is real, but its cause is the lack of compositional operator structure, not connectionism per se — and that structure is exactly what this design supplies.

Item 55 is non-interpretable, so "the abstract tokens are operators" is an inference, not an established fact — by construction we cannot confirm it directly. The design commitment (parameterize thoughts as morphisms) is defensible on independent grounds from the companion document; the item-55 connection is supporting evidence, not proof. A probing experiment that tests whether learned latent thoughts behave like composable operators is Q4.

The Block: Attention, Activation, Norm

3a   The Attention Layer see companion §3

The attention mechanism is the relational attention developed in the companion document, not repeated here: relations as a monoid acting on embeddings, the polar (magnitude = content, phase = structure) hyle/morphe split, softmax as the discrimination/classifier, expressivity from a small generating set, and SSA (item 5) handling content-conditioned routing so the structural channel stays linear. In the recurrent block it is the per-loop operator: each depth iteration applies relational attention, and over a thought step the composed attention is the composed morphism. The polar phase channel is also where the activation question below becomes concrete.

3b   TBAF — Used Once, For Drift Correction open

The agenda's earlier note on the Triangle-Based Activation Function was misleading and has been corrected (item 69). Two facts reframe it. First, in the repo experiment the activation was used only once in the entire model — an image model — and still achieved its goal; it was never required in every block. Second, and more important, its purpose was to solve autoregressive drift — the compounding error that accumulates when a model is fed its own output repeatedly. That is exactly the limit that bounds horizon in §1g. So the real question is not "replace SwiGLU everywhere" but "can one targeted placement correct the latent-rollout drift that caps how far the world model can be rolled forward?"

Why a triangle activation would correct drift. It is bounded, periodic, and (with unit slope) approximately 1-Lipschitz. Boundedness caps signal-magnitude growth; the ~1-Lipschitz property caps per-step error amplification — and drift is error amplification, so bounding the amplification factor bounds the compounding; and unlike tanh/sigmoid it does not saturate, so it controls magnitude without killing gradients. On the phase channel the argument is stronger still: phase is periodic, so accumulated phase drift wraps back into range rather than escaping — the triangle activation matches the geometry instead of fighting it.

Candidate placements, most-interesting first:

1
The phase channel of relational attention. A triangle wave is the principled nonlinearity for a periodic variable, and it doubles as drift correction — accumulated phase drift folds back onto the torus. Content (magnitude) is left untouched, which is essential: folding magnitude would corrupt content. The polar split is exactly what makes this targeted application possible. Strongest fit — it links item 69, the polar split, and the §1g horizon lever in a single placement.
2
The rollout feedback path. One activation where the predicted latent is fed back as the next input — the precise site where autoregressive drift enters. The placement most directly aligned with TBAF's original drift-correction purpose.
3
The single shared recurrent block. In a weight-shared loop, "used once" in parameter terms means "applied every iteration" in compute terms; a bounded periodic activation aids stable signal propagation across loops (the SIREN rationale), addressing the §1c stability hazard as well.

The result is from a single image model; whether the drift-correction property transfers to an abstract reasoning latent space is unproven, and it is the crux of the user's own question. The precondition is specific: TBAF helps only if the meaningful drift lives in the bounded/periodic part of the representation — phase drift (it wraps) or magnitude growth (it caps). If drift is instead semantic — meaning drifting while magnitude and phase stay bounded — a periodic activation will not fix it, and the correction must come from elsewhere (JEPA prediction-error re-grounding, or the metacognitive check). So the first experiment is not "add TBAF" but "characterize where drift lives in the latent rollout," then place TBAF only if it lives in the periodic/bounded part. The "used once" and "for drift" claims are both worth re-verifying against the repo first.

3c   Activation & Normalization Baseline established

Absent a confirmed reason to deviate, the block uses the robust modern defaults: SwiGLU FFN (item 13), RMSNorm (item 36), and QK-Norm (item 56). QK-Norm is doubly relevant here: it stabilizes attention logits in deep recurrence (the §1c stability concern) and it is the natural place to control the magnitude channel of the polar split — keeping content magnitude bounded while phase carries structure. TBAF, if it survives testing, is a single-site addition on top of this baseline, not a replacement for SwiGLU throughout.

04

Memory & Training Under 4GB

The binding constraint is 4GB of VRAM on an RTX 3050 Ti. Every architectural choice above is shadowed by it, and a nested T × L recurrence is exactly the structure that naively blows the budget. Three levers make it tractable.

4a   DiffusionBlocks — Block-wise Training directional — high value

DiffusionBlocks (item 85, Sakana) splits the network into blocks trained independently by treating the forward pass as a diffusion-style denoising process, so the memory cost is decoupled from the full unroll depth — you never need all activations of the full network resident at once. For a recurrent-depth model this is an unusually good fit: the loop is already a sequence of near-identical block applications, which is precisely the structure block-wise training wants. This is the single most promising memory lever for fitting the nested recurrence on 4GB, and it is why item 85 entered the agenda.

Whether independently-trained blocks preserve the semantics of a weight-shared recurrent loop — where the same block must compose with itself across iterations — is not obvious and is Q5. Block-wise training assumes a degree of block independence that recurrence partly violates.

item 85 (DiffusionBlocks)item 88 (PID — diffusion decoding)

4b   Implicit Gradients for the Depth Loop theoretical

If the depth recurrence converges to a fixed point h* = B(h*), the Deep Equilibrium trick computes the gradient at the fixed point via the implicit function theorem, in O(1) memory regardless of the number of loop iterations — you do not store the per-iteration activations at all. This is the clean theoretical answer to the depth half of the memory problem, and it pairs with DiffusionBlocks (which handles the across-block half). The cost is the assumption of convergence and the overhead of the implicit solve; whether the relational recurrent block converges reliably enough to use DEQ gradients is open.

item 1 (looped/equilibrium)item 41 (attractors = fixed points)

4c   Quantization, MoE, Checkpointing established

The standard levers, all compatible with the above: 4-bit weight quantization (items 60, 67) for the resident model; MoE + MLA (item 11) to grow capacity without growing active parameters or KV-cache; and gradient checkpointing as the always-available fallback. None of these is novel; they are the cushion that makes the novel parts fit.

Residual Coupling (item 65) is the cross-expert mechanism that doubles as the companion document's sheaf-gluing of perspectives — but it carries a real trade-off and is not a drop-in MoE replacement. Because it couples the experts' residual streams, it requires the experts to be active, which forfeits exactly the sparsity that makes MoE efficient. So it loses on raw efficiency versus a traditional sparse MoE. It is worth paying for only where cross-expert coupling — cross-domain reasoning, perspective-gluing — matters more than the compute savings of sparsity. Use it deliberately, not by default.

Systems & Implementation

xAI's vertically integrated training stack, written in C, reportedly trains over an order of magnitude faster than JAX through heavy cross-GPU parallelism (item 83). The lesson is real — a tightly controlled low-level stack is a genuine multiplier — but the timing matters. Rust (items 83, 84) for bare-metal speed and bloat removal is a commercialization-phase concern, not a testing-phase one. During research, a faster stack mostly buys faster iteration; it does not change which architecture is correct, and premature low-level optimization would trade the thing that is scarce (research iterations on a 4GB card, where the constraint is memory, not language overhead) for the thing that is not yet needed (throughput at scale).

Hold Rust/C systems work until the architecture is validated. The 4GB constraint is addressed by the memory levers in §4, which are algorithmic, not implementation-language choices — a faster language does not create VRAM. Revisit items 83/84 when moving from "does this architecture work" to "can this be deployed and scaled."

Build & Test Blueprint — Non-Hierarchical v1

This is the consolidation of §0–§5 into one buildable, testable model, stripped to what serves the two goals — data efficiency and reasoning depth — with the hierarchy left out (§1e, A7). The linchpin is a single idea that ties the rest together: a recurrent-depth core trained as a Deep Equilibrium model, whose equilibrium is an attractor (item 41).

6a   The Core — One Block, Settled to an Attractor (DEQ) directional

A single weight-shared block fθ — relational attention (§3a) plus RMSNorm/QK-Norm (§3c) — is iterated on a fixed input until it reaches equilibrium h* = fθ(h*, x). That equilibrium is the attractor: the settled reasoning state. This single move serves both goals at once:

Reasoning depth

  • Iterations-to-converge = effective depth, unbounded and adaptive — harder inputs settle more slowly, so the model spends more computation where it is needed.
  • Depth is decoupled from both layer count and parameter count (the fθ operator view, §1a / §2).

Data efficiency

  • DEQ implicit gradients train the loop in O(1) memory regardless of iteration count (§4b) — depth becomes affordable on 4GB.
  • Basins of attraction are denoising: nearby noisy inputs fall into the same attractor = the same conclusion, which is generalization from few examples.

Halting falls out for free: stop when the loop reaches its fixed point (‖ht+1 − ht‖ < ε), the convergence-based halting of §1f — no EOS token for the latent loop. And the same settling engine is reused by the learning algorithm: prospective configuration's relaxation is this same DEQ solve with the target clamped (learning-algorithm §5), so inference and learning share one mechanism.

DEQ gradients want a stable, unique fixed point, but reasoning may want multiplicity (breadth, alternatives) and HRM's analysis found spurious fixed points that trap the dynamics. For v1 — single-timescale, depth-only — single-attractor convergence is the right simple target, and the chain axis (§1b) is where breadth lives later. Whether the relational block reaches useful, non-spurious attractors reliably enough for DEQ is the load-bearing assumption of the whole core (A8); a contraction constraint (spectral norm, the ~1-Lipschitz placement of §3b) is the first lever if it does not.

6b   I/O — Encoders Yes, Reconstructive Decoders No directional

Because the objective is JEPA — predict in representation space, never reconstruct the input — the model needs something to map raw input into the latent space (an encoder) and something to emit (a head), but no generative/reconstructive decoder. The capacity that an autoencoder or diffusion model spends learning to repaint its input is simply not spent here. That is the data-efficiency win at the I/O layer.

The better framing than "one encoder-decoder per modality" is to treat I/O as clamping on the shared core, the way a predictive-coding network does:

1
Encode = clamp + settle. A thin per-modality adapter places the input on a set of sensory units; clamp them and let the core settle. The attractor it reaches is the encoding — no deep encoder stack.
2
Cross-modal prediction = clamp some, free the rest. Clamp one modality's units, leave another's free, settle — that is JEPA prediction in the shared space, and it is how multimodality is integrated without a separate fusion module.
3
Output = read off. A thin head reads the relevant units after settling. For text this head is autoregressive (a PrefixLM-style token head, as in HRM-Text) — the one place "settle and read off" does not suffice, since token generation is discrete and sequential.

Under this view multimodality is which units are clamped, not which network you route to; adding a modality is adding a thin adapter, not rebuilding. One shared core, thin adapters in, thin heads out.

Three caveats. Text still needs a real autoregressive token head, so "no decoder" means no reconstructive decoder, not no output head. JEPA still needs a target encoder (EMA/stop-gradient copy) and an anti-collapse term (SIGReg/VICReg, §1d) — that machinery does not disappear. And multimodal-JEPA-in-a-shared-core is research-grade (JEPA is proven mainly in vision), so build the single-modality text core first and add modalities as clamped channels later (A9). If clamped settling underperforms dedicated encoders, the fallback is thin per-modality encoders into the shared core — still no reconstructive decoders.

6c   Training for Data Efficiency directional

Five compounding sources of data/compute efficiency, none of which require the hierarchy:

Weight-sharing
One block reused for all depth — effective depth at the parameter cost of a single block (§1a).
DEQ O(1) memory
Implicit-function-theorem gradients at the fixed point; no per-iteration activations stored (§4b).
Attractor denoising
Basins generalize many inputs to one conclusion — robustness from few examples.
Deep-supervision outer loop
HRM's data-efficiency device: repeated supervised segments squeeze signal from little data — the part of HRM that matters in the small-data regime Ouro's 7.7T-token recipe does not need.
JEPA latent objective
No capacity spent reconstructing inputs; the prediction target lives in representation space (§1d).

The learning rule for v1 is the DEQ implicit-gradient backprop above — standard and known to work. The research upgrade is to replace it with prospective configuration reusing the same settling engine, filtered by item 52's signal/noise decomposition (the entire learning-algorithm companion document); that is staged after the architecture is validated, not built in from the start.

Open Architectural Questions

A1 Do depth recurrence and thought recurrence co-train in one model? open — central

The orthogonality argument says nesting COCONUT (thought) around Ouro (depth) should be possible; it does not prove gradient descent finds a joint solution through a T × L unroll. The first experiment: train depth-recurrence alone to a working baseline, then add the thought loop and test whether joint training is stable and whether the combination outperforms either axis alone. Fallback is graceful — ship depth-only if nesting fails.

A2 Can a JEPA objective drive the latent chain without collapse, through a depth-recurrent core? open — highest risk/reward

A3 Can a targeted TBAF placement correct latent-rollout drift and thereby extend horizon? open

TBAF's purpose was autoregressive-drift correction, which is exactly the limit that bounds horizon (§1g) — so this is not merely an activation-choice question but a horizon-extension one. First characterize where drift lives in the latent rollout (phase drift, magnitude growth, or semantic drift); then, only if it lives in the bounded/periodic part, test the placements from §3b — phase channel (strongest fit), rollout feedback path, shared block — against a SwiGLU baseline, measuring achievable rollout length before divergence, not just task accuracy. The open risk is transfer: the result is from an image model, and semantic drift would be immune to a periodic activation. Re-verify the "used once" and "for drift" claims against the repo first.

A5 Does DiffusionBlocks' independent-block training preserve recurrence semantics? directional

Block-wise independent training assumes a degree of block independence that weight-shared recurrence partly violates — the same block must compose with itself across iterations. Whether DiffusionBlocks' memory savings survive the recurrent setting, or need modification, is the key feasibility question for fitting the nested model on 4GB.

A6 How should a recurrent-depth model handle long context — and is the context window the wrong primitive? open

Context rot in long windows (item 87) may be a symptom of treating context as a flat buffer. A relational, recurrent model suggests an alternative: context as relational structure the model composes into, with RAG-style retrieval informing what is loaded rather than a monotonically growing window. A8 Does the relational recurrent block reach useful, non-spurious attractors reliably enough for DEQ? open — linchpin

The whole §6 core assumes the loop converges to a stable, meaningful fixed point. DEQ gradients require it; HRM's analysis found spurious fixed points that trap the dynamics; and reasoning may eventually want multiplicity the single-attractor view forbids. Measure convergence behavior on the relational block: does it settle, how often to a spurious attractor, and do basins align with correct answers? If convergence is unreliable, impose a contraction constraint (spectral norm / the ~1-Lipschitz placement of §3b) or fall back to a fixed iteration count with truncated backprop. This is the single assumption the buildable core most depends on.

A9 Can shared-core clamped settling replace per-modality encoder–decoders? open — deferred

The §6b proposal treats I/O as clamping on one shared core — encode by clamping and settling, predict cross-modally by clamping some modalities and freeing others (JEPA), read off through thin heads, with no reconstructive decoder.
