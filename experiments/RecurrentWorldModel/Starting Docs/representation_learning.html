The Core Problem

open

A model trained on ten million cat images does not learn what a cat is. It learns a high-dimensional statistical summary of what cat images looked like — which encodes the cat, the background, the typical lighting, the framing conventions of animal photography, and the JPEG compression artifacts in the dataset. Gradient descent has no mechanism that distinguishes a causal feature from a spurious co-occurrence. Both reduce loss equally, so both get encoded.

The pixel-correlation problem is not a domain-shift problem. A model can fail to learn "cat as cat" even when evaluated in exactly the training distribution — it has simply encoded the wrong features with high confidence. PMH-style deployment robustness and standard augmentation do not address this.

This is shortcut learning in its most fundamental form. The model finds any feature that minimizes training loss — if beige background predicts "cat" in the training set, the model will use it. The literature frames this variously as spurious correlations, texture bias, Clever Hans phenomena, and OOD generalization failure. All names for the same problem: the gradient signal rewards prediction accuracy, not conceptual correctness.

The root cause is that natural text and images lack the structural grounding that makes intermediate reasoning states checkable. In mathematics and code, every step can be verified: a proof step is either valid or invalid, code runs or doesn't. In medicine, law, biology, and most of the world, intermediate reasoning steps are narrative and ambiguous. There is no ground truth to check intermediate steps against — until you construct one, which is what a knowledge graph provides.

Shortcut learning (the core problem here) is distinct from domain shift (what PMH addresses). Shortcut learning means the model encoded the wrong feature reliably. Domain shift means the right feature was encoded but the distribution changed. Both cause failure but have different solutions.

1a   Perturbation Matching Hypothesis deployment tool

PMH — Perturbation Matching Hypothesis
12/13 blocks pass unified theory

A geometric theory that unifies CORAL, DANN, augmentation penalties, and metric learning as different estimators of the same object: Σ_task, the covariance of label-preserving deployment nuisance. The core claim: estimate Σ_task, penalize the encoder's Jacobian in that direction, and representation drift under deployment shift is suppressed. Seven estimator types (D1–D7) cover sensor noise, domain shift, temporal drift, style, adversarial perturbations, and compositional coordinate change.

Architecture
80M GraphMERT encoder-only (library is framework-agnostic: PyTorch, sklearn, HuggingFace)

12/13 paper blocks pass pre-registered criteria; Office-31 fails with documented eigengap reason
Scope
Label-preserving deployment change (new camera, new site, new domain, temporal drift)
Explicit exclusion
Spurious correlations (Colored MNIST, Waterbirds) — label-correlated nuisance is out of scope
Verdict for core problem: Does not address shortcut learning. PMH assumes the model encoded something approximately correct and protects it at deployment. It is downstream of good representations, not a source of them. Most useful as a fine-tuning / post-training robustness tool.

1b   LeWorldModel / JEPA directional

LeWorldModel (LeWM) — Maes, Le Lidec, LeCun, Balestriero et al. 2026
48× faster planning physical structure

The first JEPA trained stably end-to-end from raw pixels with two loss terms: an MSE temporal prediction loss and SIGReg, a regularizer that forces latent embeddings to be isotropic Gaussian by projecting them onto random directions and minimizing the Epps-Pulley normality test statistic along each projection. By the Cramér-Wold theorem, matching all 1D marginals forces the full joint distribution to be Gaussian. No stop-gradient, no EMA, no frozen pretrained encoder.

SIGReg is a geometric constraint, not a semantic one. It says the representation should be spread out isotropically in all directions. It says nothing about what those directions represent. A model satisfying SIGReg perfectly can still encode "fur texture" and "indoor lighting" as its primary axes. The temporal prediction signal pushes back against this in action-conditioned settings, but only to the degree causally irrelevant features actually vary unpredictably across frames.

Verdict for core problem: Directionally correct. Temporal prediction over diverse actions is an anti-shortcut bias; features irrelevant to next-state prediction will generate confusing gradients. But "predictable" ≠ "conceptually correct," and there's no guarantee about which features dominate. SIGReg is worth adopting as the anti-collapse mechanism.

1c   Knowledge Graph Approach (Princeton group) strongest signal

KGs as Implicit Reward Models — Kansal & Jha, Princeton 2025
beats GPT-5.2 process supervision

A bottom-up learning paradigm: ground models in axiomatic domain facts (KG triples), then use the graph's multi-hop paths as a process supervision reward during RL fine-tuning. The KG provides both the training curriculum (SFT examples derived from 1–3 hop paths) and the verifiable reward signal (path alignment: does the model's chain-of-thought mention the correct intermediate entities?). SFT instills atomic facts; GRPO with path-alignment reward teaches composition.

Theoretical Grounding

2a   Platonic Representation Hypothesis directional

Huh, Cheung, Wang & Isola (2024) argue that neural network representations are converging toward a universal statistical model of reality as models scale on diverse data. Vision and language models, trained on entirely different modalities with different objectives, show increasing representational alignment as they grow larger. The hypothesis is that this convergence is directional — it is approaching the true causal/generative structure of the world, the joint likelihood function across all modalities and tasks.

The ideal representation exists independently of any particular model, modality, or training run. All models are asymptotically approaching it. This is the Form — what all models are converging toward as they scale and diversify. The representation is uniquely determined by the structure of reality, not by any particular dataset or objective.

Implication for this agenda: scale + diversity on temporally-structured data (LeWM objective) is one valid path toward correct representations. The convergence pressure from diverse environments averages out spurious correlations, leaving only causal features as the invariant signal.

Honest limitation: Convergence may be directional without being fast. Models are better shadow-watchers than they were five years ago while potentially still encoding shadows as primary representations. The theory says nothing about how many steps remain to the limit.

2b   Plato, Socrates, and the Cave theoretical

Plato's diagnosis: Training data are shadows on a cave wall. The model watches shadows carefully, learns patterns in how they move, and mistakes this for knowledge of the world. Individual cats in any image are imperfect instantiations of the Form of Cat — the essential, invariant structure. You cannot derive the Form by accumulating more shadows. No amount of additional training data closes the gap.

Plato's prescription: True knowledge of the Form requires a different epistemic move. In architectural terms: a structural prior — a priori knowledge of the Form baked into the learning system. This is item 34's argument. Algebraic structure embedded as geometric transformations is, in Platonic terms, injecting the Forms directly into the representation space.

Socrates' test: The model knows millions of cat examples but cannot define one. Socrates' method — elenchus — generates counterexamples that destroy false definitions. In ML terms: adversarial probing, OOD evaluation, interventional testing. The question is never "does the model get the right answer" but "does it get the right answer for the right reasons."

Socratic test for representations: give the model a cat on a black background; a cat-shaped sculpture that isn't a cat; a domestic cat vs a wild cat with different color statistics. If confidence in "cat" covaries with background texture rather than object structure, you don't have a definition — you have a catalogue of typical examples.

2c   The Two Paths

Empiricist path (Platonic Rep. Hyp.)

Diverse data + temporal prediction → spurious correlations average out → causal features survive as invariant signal → Platonic convergence. Scale and diversity are sufficient; the right representations emerge.

Mechanisms: LeWM temporal prediction; active inference (item 20); multi-environment IRM-style training.

Weakness: Plato would say you cannot reach the Forms through induction alone. Convergence may be slow or plateau at a level still far from the correct concept.

Rationalist path (KG algebra / item 34)

Inject explicit algebraic structure of the domain as a prior. Embed logical rules as geometric transformations. Make the Form visible to gradient descent by constructing it externally and supplying it as training signal.

Mechanisms: KG reward model (item 14); GraphMERT extraction; group-theoretic training (item 7).

Weakness: Requires domain structure to be formalized. Auto-KG factory partially solves this but cannot yet bootstrap from zero.

Both are probably necessary. The empiricist signal (temporal prediction, diverse environments) provides the training pressure that aligns representations with reality. The rationalist scaffold (KG-derived reward, algebraic structure) makes the target visible to gradient descent before convergence has occurred naturally. Neither alone is sufficient at tractable scale.

2d   Aristotle: Immanent Forms, Four Causes, Predicables theoretical most concrete

Aristotle's foundational disagreement with Plato is the one most consequential for architecture. Plato places the Forms in a separate transcendent realm — physical things merely participate in them. Aristotle says the Form of a cat is not elsewhere: it is the actual organizational structure of the cat, present in it, readable from it by a correctly performed abstraction. This is neither pure empiricism (scale will surface it) nor pure rationalism (inject it from outside). The Forms are immanent — they are in the data. The problem is extracting them correctly.

Hylomorphism and the telos insight. Every substance compounds matter (hyle, substrate) and form (morphe, organizational principle). For a chair: matter is wood, metal, or plastic; form is whatever structure makes it a chair. The key insight: the form of all chairs is constituted primarily by their final cause — their telos, which is for sitting. All variety of material and structural form is compatible with being a chair; what is invariant across every instance is the function. Hylomorphism and telos are mutually grounding: form is the structure that realizes telos in matter.

Item 68 independently discovered a rudimentary version of this. The cosine head attends to material features — content, surface statistics. The sine head (Gram-Schmidt orthogonalized queries) attends to structural pattern — positional relations, independent of content identity. It works on multiplication because arithmetic structure is morphe and digit identity is hyle. But the split is not robustly enforced: the fusion weight α is a single learned scalar per layer with no invariant maintaining the separation; both streams share one loss; the sine head has no external guidance on what "structure" means beyond what arises from the arithmetic task. In an arbitrary domain the split would dissolve. The robust resolution is not a stronger penalty but a better coordinate system: the polar decomposition (§3b), where content is magnitude and structure is phase. These are independent coordinates, so a structural relation that rotates phase leaves content exactly invariant — robustness by construction rather than by regularization.

The Four Causes. A complete representation answers four causal questions. Current models are trained almost exclusively on the first:

Material cause — over-encoded
What it is made of. Pixels, surface statistics, substrate. Highly instance-specific and domain-local. The dominant feature in current models — the root cause of shortcut learning.
Formal cause — partially addressed
What kind of thing it is. Organizational structure, type, categorical identity. The KG's genus/species/differentia structure addresses this. Should be the primary encoding target.
Efficient cause — under-encoded
What mechanism or agent produced it. Generative process, causal history. More generalizable than surface form. Largely absent from current training signals.
Final cause (telos) — most neglected, most important
What it is for. Function, purpose, role. The most invariant property across all instances — all chairs share the same telos regardless of material or exact structure. Neglected everywhere in current ML.

Predicables as KG edge types. Aristotle's five predicables classify how predicates relate to subjects — a direct taxonomy for weighting the path-alignment reward. The fundamental insight: training the reward signal on accident edges as if they were differentia edges is the spurious correlation problem stated with precision.

DIFFERENTIA Highest reward. What distinguishes this species within its genus — whose presence/absence determines which kind of thing something is. Encodes the formal cause. Training on differentia edges builds essential-feature representations.
GENUS High reward. Broader category membership. Enables hierarchical compositional reasoning and analogy within a category.
PROPERTY Moderate reward. Necessarily true of all instances but not definitional. "Cats purr" — reliable inference signal, but not what makes something a cat.
ACCIDENT Suppress or zero. Contingently true of some instances. "This cat is orange." These edges must not appear in path-alignment reward paths — they encode exactly what we are trying to prevent the model from learning.

Each concept node in the unified KG carries four dedicated embedding slots: material, formal, efficient, and telos. KG edges are typed by the cause they encode in addition to their predicable classification. Path-alignment reward weight is a joint function of both: formal/telos edges with differentia predicable type carry maximum weight; material/accident edges carry zero. This transforms the KG from a taxonomy into a full causal model and gives each representation its complete Aristotelian profile.

2e   Kant, Hegel, Heidegger theoretical

The Greek philosophers identify what correct representations should be and how to extract and validate them. The German tradition asks what must be architecturally given for correct representation to be possible at all, and what dynamics govern its development. This maps onto item 50's tripartite structure — Epistemology, Ontology, Methodology — each completed by one German philosopher in a way the Greeks did not reach.

Kant — Epistemology: Conditions for Possible Knowledge. Experience is structured by a priori forms of intuition (space, time) and twelve pure categories of understanding that are not learned from experience but are what makes experience possible at all. The architecturally critical ones: substance/accident, causality, community (mutual interaction), and the modal triad of possibility, actuality, and necessity.

The strongest claim in this analysis: the substance/accident distinction (morphe/hyle), causal structure, and modal status cannot be learned by gradient descent — they are conditions for gradient descent to learn anything correctly at all. They should be architectural priors. A model without a substance/accident prior can learn correlations but cannot distinguish essential from accidental features, because making that distinction requires a framework that precedes any particular training signal.

Kantian modality extends KG edge typing with a dimension orthogonal to Aristotle's predicables, giving a 5×3 edge classification matrix — five predicable types crossed with three modal statuses, each cell with a principled reward weight:

Modal KG edge types

  • Necessary: holds in all possible instances (triangles have three angles). Highest reward. Encodes what cannot be otherwise.
  • Actual / Contingent: holds in fact but could be otherwise (mammals typically have four limbs). Moderate weight.
  • Possible: could hold; doesn't necessarily. Low weight; reserved for counterfactual reasoning tasks.

Combined 5×3 matrix

  • Differentia + Necessary → maximum reward weight
  • Property + Actual → reliable inference signal
  • Accident + Possible → suppress entirely
  • 15 distinct edge classes with graduated reward weights, derived from first principles

The transcendental unity of apperception — the binding condition that makes all representations belong to a unified experiencing subject — maps onto metacognition (item 80). For a COCONUT latent model, the HRM slow controller is the "I think" that accompanies and binds the latent reasoning chain. Without this, the model has disconnected feature activations but not integrated knowledge.

Hegel — Methodology: Dialectical Development. Every determinate concept contains its own negation; their confrontation produces an Aufhebung (sublation) — a synthesis that preserves what was true in both while ascending to a higher level. The EM loop is Hegelian in structure: current KG (thesis) → model's OOD failures (antithesis) → M-step revision (synthesis). Each cycle produces a richer KG than either the original formalization or the raw model learning could generate alone.

Hegel also gives the correct model for curriculum design. Each stage should generate the specific contradictions that specify the next stage — not arbitrary progression but necessary development:

1
Sense-certainty: single-hop within-domain paths. Masters immediate concept association. Internal limitation: cannot compose.
2
Perception: 2–3 hop within-domain paths. Learns compositional patterns. Internal limitation: cannot transfer structure across domains.
3
Understanding: cross-domain paths via bridge edges. Structural isomorphisms become visible. Internal limitation: can traverse bridges but cannot reason about why bridges exist or generalize the bridging principle.
4
Reason: meta-structural analogies — reasoning about the form of reasoning itself. Generalization over graph structure, not just within it.

Heidegger — Ontology: Being-in-the-World. The distinction between Zuhandenheit (ready-to-hand) and Vorhandenheit (present-at-hand) sharpens the telos argument. A hammer in use is transparent — you hammer without attending to the hammer. It becomes present-at-hand only when it breaks. Ready-to-hand is the primary ontological mode; present-at-hand (descriptive, theoretical) is derived and secondary. Current models operate almost entirely in Vorhandenheit: they describe and classify. JEPA and active inference objectives produce Zuhandenheit representations — organized around what things do and what can be done with them, not what they look like.

Training objectives are ontologically constitutive, not measurement tools. A model trained with next-token prediction inhabits a world of sequential text. A model trained with JEPA inhabits a world of state transitions and physical consequences. These produce qualitatively different representations because they structure the world differently. The unified KG is the world the model is thrown into — its ontological home, pre-shaping all representations before any training example is encountered.

2f   Item 50: The Complete Framework synthesis

Epistemology, Ontology, and Methodology are the exact tripartite structure needed to organize the six-philosopher synthesis. Each dimension is addressed incompletely by one Greek philosopher and completed by one German philosopher:

Ontology — what there is

Plato: transcendent Forms as targets; KG as formal scaffold

Heidegger: Forms are organized by practical concerns; Zuhandenheit; training objectives constitute the world

→ Unified KG organized around telos and function, not only taxonomy

Epistemology — how we know

Socrates: dialectical testing; elenctic validation of genuine knowledge

Kant: a priori categories as architectural priors; 5×3 modal-predicable edge matrix; apperceptive binding

→ Polar hyle/morphe split (magnitude/phase, §3b) + softmax-as-classifier for exclusion + adversarial OOD validation; the 5×3 matrix collapses to a small lattice (§3c)

Methodology — how we arrive

Aristotle: immanent Forms; hylomorphic extraction; predicable-typed edges; four-cause embeddings

Hegel: dialectical Aufhebung; EM loop as thesis→antithesis→synthesis; four-stage phenomenological curriculum

→ Predicable+modal typed EM loop with Hegelian curriculum stages

Plato supplies the formal target but not the extraction method. Aristotle supplies extraction but not the conditions. Kant supplies the conditions but not the target or dynamics. Hegel supplies the dynamics but not the target or conditions. Heidegger grounds the ontology but not the validation. Socrates validates but does not construct. Each is necessary for the others to function. But the whole synthesis rests on one shared assumption — representationalism, that intelligence is building a correct internal model of a world of fixed Forms — which the Eastern traditions contest at the root (§2g), and which category theory then reconciles (§2h).

2g   Eastern Critiques: Emptiness, Exclusion, Perspective theoretical critique + extension

Before the individual thinkers, one structural point. The entire Western synthesis (§2a–2f) is representationalist: it assumes intelligence consists in building a correct internal representation of an external world. Even Heidegger, who attacks the subject–object split hardest, still aims at being-in-the-world correctly. Most major Eastern traditions challenge this at the root — some deny there is a fixed external structure to represent, some deny the knower/known distinction, some hold that the highest intelligence is non-representational skill. That is the deepest challenge they pose, and it runs beneath every specific contribution below.

Nāgārjuna (Madhyamaka) — emptiness against fixed Forms. Śūnyatā: all phenomena are empty of inherent self-nature (svabhāva). Pratītyasamutpāda (dependent origination): everything arises in dependence on other things. "Cat" has no intrinsic catness; it is an empty designator for a cluster of interdependent relations — remove the contrast class, the perceptual conditions, the linguistic community, and it shifts. This is a direct refutation of the Platonic Forms and of the Platonic Representation Hypothesis's convergence claim: if there are no fixed natures, there is no fixed target to converge toward. Crucially it is not nihilism — what exists is a network of relations without fixed nodes; the relations are primary, the nodes are merely where relations cluster. Architectural consequence: KG nodes should carry no hardcoded content separate from their edges. Node identity is relational profile; nodes can be bootstrapped from edges, and split, merged, or dissolved by the learning process. (§2h shows this is exactly the Yoneda Lemma.)

Dignāga (Yogācāra) — apoha, concepts as exclusion. The apoha theory: concepts have no positive universal content; "cat" means "not non-cat." A concept is a boundary, not a content — you learn it by accumulating successful exclusions, not by abstracting a shared positive essence. This is a genuine alternative to Aristotle's predicables and connects directly to contrastive learning and the Socratic elenchus (both proceed by elimination). The caution flagged earlier stands: the strong claim that there is no positive content is probably too strong, and over-literal application risks an explosion of negative edges. The resolution is mathematical, not a compromise — see the subobject classifier in §2h. The defensible upshot for training: reward discrimination/exclusion at least as much as positive identification.

Jain (anekāntavāda) — irreducible perspectivalism. Reality is many-sided; no single standpoint grasps the whole. Syādvāda qualifies every judgment with "perhaps, from standpoint X"; the nayas are those standpoints. This extends Kantian modality: whether a relation is necessary or contingent is itself standpoint-relative ("triangles have three angles" is necessary Euclidean, contingent otherwise). The 5×3 predicable×modal matrix becomes 5×3×perspective — which sounds like an explosion until §2h shows it is handled by functors and sheaf-gluing. This is also the cleanest resolution of the unified-vs-specialized model debate (§6): the generalist and the specialist are each partially valid from their standpoints, so the right move is to maintain multiple standpoints simultaneously — which is precisely what the Residual-Coupled MoE does.

Daoism (Zhuangzi, Laozi) — natural joints and wu wei. "The Dao that can be named is not the eternal Dao." Zhuangzi's cook carves the ox by following its natural joints rather than imposing cuts; he meets no resistance because he attends rather than forces. The critique of the KG project: any ontology imposes a vocabulary — how do you know your cuts follow reality's joints rather than your domain experts' habits of speech? Wu wei (effortless, non-forcing action) as a training principle favors objectives that let structure emerge (JEPA, active inference) over supervised classification against fixed labels. And over-specified systems are brittle: the more precisely the KG is pinned down, the more out-of-distribution failures its categories will generate. The KG should be a light scaffold that dissolves as the model's own joints develop.

Advaita Vedānta (Shankara) — superimposition is shortcut learning. Adhyāsa (superimposition): the false attribution of one thing's properties to another — seeing a rope and perceiving a snake. This is the shortcut-learning problem in a different vocabulary: the properties of a typical background superimposed onto the concept. The solution is viveka (discrimination), treated as a dedicated faculty for separating the real from the superimposed — distinct from perception and inference. Architecturally this suggests a metacognitive module whose specific job is to detect when a prediction is driven by accidental co-occurrence rather than essential structure: not uncertainty quantification (confidence in the answer) nor OOD detection (distribution shift), but a check on which features are doing the work.

1 · Anti-svabhāva (Nāgārjuna): there may be no unique correct target — challenges PRH convergence. 2 · Apoha (Dignāga): exclusion may matter as much as positive content — redirect part of the reward toward discrimination. 3 · Wu wei (Zhuangzi): imposed structure is brittle — keep the ontology fluid and revisable. 4 · Perspectivalism (Jain): no single representation is correct from all standpoints — maintain multiple standpoints rather than forcing one. The common demand: treat the ontology as a hypothesis about reality's joints, not a description of its eternal Forms.

2h   The Category-Theoretic Bridge: Nāgārjuna Meets Yoneda theoretical grounds §3

The deepest Eastern critique — Nāgārjuna's denial of fixed natures — looks like it should reject the whole representational enterprise. Category theory shows the opposite: it formalizes the critique and thereby strengthens the framework. The Eastern objections are not obstacles to route around; they are the informal statement of theorems that license the architecture in Section 3.

Yoneda Lemma = dependent origination. The Yoneda Lemma states that an object A is completely determined, up to isomorphism, by Hom(−, A) — the totality of morphisms into it from every other object. Identity is relational profile. This is Nāgārjuna stated mathematically: no svabhāva (no intrinsic content), yet objects still exist — as their web of relations. It dissolves the apparent Plato/Nāgārjuna conflict: the Forms are real (objects exist) but carry no intrinsic content (their identity is exhausted by relations). It also sharpens the Platonic Representation Hypothesis into something falsifiable: models are converging toward the Yoneda embedding of the concept-category, not toward a vague "statistical model of reality." This is exactly the foundation the CTKG already builds — Yoneda grounding, with KL divergence as the graph growth/stability signal.

Subobject classifier = apoha, with positive content preserved. This resolves the Dignāga tension precisely. In a topos there is a subobject classifier Ω: subobjects (membership) correspond to morphisms into Ω. Exclusion is then the complement of the positive morphism structure — recovered from positive content via Ω, never stored as O(n²) explicit negative edges. So both intuitions are right at once: the morphisms are the positive content (Dignāga's "no content" is too strong), and exclusion (apoha) is derived from them as a complement. Discrimination comes for free from positive structure plus a classifier — which in the architecture is what softmax normalization does (§3c).

Partial orders + sheaves = the dimensional resolution. The fear of exploding dimensionality (5×3 matrices, perspectival blow-up) dissolves once the structure is named. Predicables (differentia > property > accident) and modality (necessary ≥ actual ≥ possible) are partial orders — small lattices; their product is compact, and reward weighting is a monotone map on it, not fifteen independent cells. Perspectives (Jain nayas) are functors between sub-categories, and the sheaf condition glues locally-consistent perspectives into global structure — the "whole truth" no single naya captures is the global section of the sheaf. Low-dimensional locally, expressive globally. This is the CTKG's sheaf-theoretic type unification, and it is why the Section 3 architecture can be both expressive and compact.

Biological realization. The structure is not merely mathematically available; it is how the brain appears to work. Entorhinal grid cells implement a metric Yoneda-style embedding — objects defined by relational position in a learned reference frame. Cortical columns (Thousand Brains) hold multiple partial models of one object and reconcile them laterally: a biological sheaf of nayas. The hippocampus, under complementary learning systems, is a category constructor — fast episodic encoding feeding slow neocortical extraction of the morphism structure. Sparse distributed representations are sparse approximate Yoneda profiles. The prefrontal cortex applies relational structure to fresh content — a functor applicator. The framework is recovering, artificially, a structure evolution already found.

This bridge is the warrant for Section 3. The architecture there — relations as a monoid acting on embeddings (Yoneda profile generated by an algebra), the polar hyle/morphe split, softmax as subobject classifier, expressivity from a small generating set, sheaf-gluing for perspectives — is the computational realization of these formalizations. The path is: Eastern critique → category-theoretic formalization → architecture. The critiques are not rejected; they are built in.

The Architectural Plan: Relational Attention

The requirements are strict: no external symbolics at inference, ≤ O(n²) with a credible linear path, and genuine mathematical grounding rather than categorical decoration. An earlier sketch named four attention heads after the four causes — but naming a head "efficient cause" does not make it one, and its proposed structural mechanism (Gram-Schmidt orthogonality) is the same fragile, dissolvable constraint we criticized in DHA. The grounded version starts from what the Yoneda Lemma actually says and builds the mechanism from there. It also lands squarely on prior work: PoPE, the CTKG, and item 7.

3a   Relations as an Algebra Acting on Embeddings core

Standard attention collapses every relation between two tokens into a single bilinear score qᵀk. But Hom(A,B) in a category is generally a set of distinct, typed, composable morphisms — not a scalar. The honest generalization: a relation type r is a transformation M_r acting on the embedding space, where "A stands in relation r to B" means b ≈ M_r·a. This is the structure behind relational and knowledge-graph embeddings (TransE = translation group, RotatE = rotation group) and, crucially, behind RoPE — which is exactly a rotation M_θ encoding the positional relation. PoPE generalizes the encoding; this generalizes the relation.

The payoff is composition. If r ↦ M_r is a monoid homomorphism, then composing relations is composing transformations: M_{r₂}M_{r₁} = M_{r₂∘r₁}. Multi-step reasoning becomes transformation composition — a genuine algebraic operation, not a metaphor. An object's identity is then its orbit under the relation monoid: the full profile of how every relation connects it to every other object. Two objects with the same orbit-structure are isomorphic. This is the computational analogue of the Yoneda Lemma made precise — identity is relational profile, and the profile is generated by an algebra, not stored as a table.

Real relations are not all invertible ("is-a" has no inverse), so the structure is a monoid (or a small category), not a group. And r ↦ M_r is only a faithful representation if training makes it one — composition fidelity must be encouraged and measured, not assumed. Whether the learned transformations actually compose correctly is the central empirical question (Q1).

An important caveat that keeps this project honestly bounded. Joint-embedding architectures (JEPA) already learn coherent object representations and causal physics on their own — they reliably distinguish a cat, a hand, or a tree from co-occurring background pixels, without conflating object and backdrop. So the perceptual half of the shortcut-learning problem (§0) is largely solved by JEPA already. Relational attention is therefore not aimed at perception; it targets the compositional layer that sits on top of JEPA-grounded objects — the algebra of relations among already-coherent objects. The division of labor: JEPA supplies the objects (the "what"); relational attention supplies the reasoning over them (the "how they relate"). Do not oversell the mechanism as solving object coherence — that is borrowed from JEPA, and the novel burden of proof is on composition.

3b   The Polar Hyle/Morphe Split robust by construction

The robustness problem DHA could not solve has a clean resolution in the coordinate system itself. Write an embedding component in polar form z = ρ·e^{iφ}. Magnitude ρ and phase φ are genuinely independent coordinates — a relation that acts by rotating phase (φ → φ + θ_r) leaves magnitude exactly invariant, by the structure of the polar decomposition, not by a learned penalty that can dissolve. Assign content (hyle) to magnitude and relational structure (morphe) to phase. Relations are phase rotations; content is what survives them.

This is item 3 (polar coordinate PE, "separates what from where") generalized from position to semantic relation, and it is the rigorous version of the hyle/morphe split that DHA only approximated. The "where" (phase) carries structure; the "what" (magnitude) carries content. Item 29 (TBT-inspired evolution on PoPE) is the same instinct. The robustness DHA lacked comes from using a real coordinate system instead of a soft orthogonality penalty.

In RoPE the inner product mixes magnitude and phase (it picks up cos(φᵢ−φⱼ) terms), so "content = magnitude, structure = phase" is the principle, not an automatic property of any rotary scheme. Realizing a clean split requires a parameterization that keeps the channels separable — distinct magnitude and phase pathways, or restricting relations to the phase torus. This is a concrete design problem, not a solved one (Q11).

3c   Discrimination and Expressivity Without Explosion resolves apoha + dimensionality

Exclusion (Dignāga's apoha) for free. We do not need negative edges. Softmax normalization is inherently contrastive — it computes "this against all others." It plays the role of the subobject classifier Ω: membership and exclusion are read off from where probability mass concentrates, derived entirely from the positive relational scores. Discrimination comes from positive structure plus normalization, with no O(n²) blow-up in explicit negative relations. Boundaries are learned; they are not stored.

Expressivity from a small generating set. The dimensionality fear — 5×3 predicable×modal matrices, perspectival blow-up — dissolves once relations form an algebra. A monoid generated by k generators reaches a combinatorially large set of composite relations by composition; you learn k generator-transformations, not the full relation space. Predicables and modality are partial orders (small lattices); their product is small, and the reward weighting is a monotone map on that product, not 15 independent cells. Perspectives (Jain nayas) are functors between sub-categories, glued by the sheaf condition — local and low-dimensional, exactly the CTKG sheaf-theoretic type unification already developed. Expressivity lives in the closure of a few generators, not in a giant explicit tensor.

Open Questions

Q1 Does the learned relation map r ↦ M_r stay a faithful, composition-preserving representation? open — central

The entire relational-attention plan rests on relations behaving as a monoid: composing two relation-transformations must yield the transformation for the composite relation (M_{r₂}M_{r₁} = M_{r₂∘r₁}). Nothing guarantees gradient descent produces this — without pressure, the learned M_r can become an arbitrary set of matrices that happen to fit local data but do not compose. The question is what training signal enforces composition fidelity, and how to measure it (e.g. holding out composite relations and testing whether the composed transformation predicts them). If this fails, the Yoneda/algebra grounding collapses to ordinary attention with extra steps. This is the first experiment to run and the one most likely to falsify the approach.

Q2 Does the EM loop converge reliably, or to degenerate solutions? directional

Theory suggests path-alignment reward prevents the worst degenerations (model must use correct intermediate entities, not just reach right answer). Algebraic consistency checking in the M-step prevents contradictions. But empirical verification on a real domain loop has not been done. This is a core experiment to run early.

Q3 Is DSS (specialized small models) necessary, or can one general model acquire domain structure? position: general model

The Princeton group concludes: separate domain experts. This project's position: that conclusion does not follow from the evidence. Item 14 proves KG-structured process supervision produces compositional reasoning. It does not prove specialization is the mechanism. A general model trained on multi-domain KG rewards should acquire equivalent domain depth while retaining cross-domain reasoning capacity that fragmented models categorically cannot have. The residual empirical question is whether a single training run can learn domain-specific KG structures without interference — which the Residual-Coupled MoE architecture (Section 6) addresses by giving each domain preferential expert modules within a unified model rather than isolated separate models. The open question is no longer whether to generalize but how to implement multi-domain KG training without domain interference.

Q4 What is the right granularity for KG nodes? directional

Too coarse (e.g., "animal") = no useful compositional signal. Too fine (e.g., "specific patient case") = a lookup table, not a generalizable graph. The right granularity is probably at the level of domain concepts: natural kind terms, functional categories, theoretical constructs. GraphMERT's ValidityScore measures ontological alignment but not granularity quality directly. This connects to the entity extraction problem in the auto-factory and to the representation learning problem itself — the right KG nodes are the Platonic Forms of the domain.

Q7 Can cross-domain KG paths be automatically discovered and used for training? open

Gaps in Current Agenda

The following approaches directly target the core representation problem but are not currently represented in the research agenda:

Object-Centric / Slot-Based Representations missing from agenda

Architectures that force the encoder to factorize observations into discrete object-like slots (SAVi, SLATE, DINOSAUR). Creates a structural prior that the world is composed of objects, which are the causally relevant units. Most direct architectural approach to preventing the "encode the background" failure mode. If the architecture can only represent N discrete entities and their relations, it can't accidentally encode irrelevant texture as a primary dimension. Should be considered alongside item 9 (manifold-constrained hyperconnections) and item 57 (single-model multimodality) as a structural prior on the encoder.

Invariant Risk Minimization (IRM) and descendants missing from agenda

Train on multiple environments where spurious correlations vary but causal features don't. Penalize features that require different predictors across environments. This is the empiricist complement to the rationalist KG approach: instead of injecting the correct structure, you force the model to discover invariant features by making the spurious ones inconsistent across environments. The PMH is a partial cousin but scoped to deployment shift; IRM targets spurious correlations directly. The catch is constructing environments where only spurious correlations vary, which can be as hard as the original labeling problem.

Add object-centric representations and multi-environment invariance training as new items to the research agenda. The former is an architectural constraint; the latter is a data-collection and training-objective strategy. Both are more directly targeted at the core problem than anything currently on the list except items 14, 34, and 7. Additionally, the six-philosopher synthesis (Section 2d–2f) introduces two concrete new tools not yet in the agenda: Aristotelian predicable-typed + Kantian modal KG edge classification (the 5×3 matrix) as a principled reward-weighting scheme, and four-cause structured embeddings as an architectural proposal. These should be tracked as research items.

Against Fragmentation: The Case for Unified Architecture

project position

The Princeton group's evidence establishes that KG-grounded process supervision produces compositional reasoning that neither pretraining nor answer-level RL can achieve. Their conclusion — build separate domain-specific models — does not follow. This section argues the opposite: that their results indicate precisely how to build domain-general intelligence, and that the DSS society architecture they propose is a strategic error that severs the cross-domain connections that characterize genuine intelligence.

6a   Reinterpreting Item 14 resolved

What item 14 demonstrates is this: when a model is given explicit compositional structure via KG path rewards, it generalizes from short-hop to long-hop reasoning in a domain it was never trained on at that depth. The variable that produced the result was the KG reward signal, not the model's domain specificity.

A general model trained on multi-domain KG rewards should acquire equivalent compositional depth across all domains simultaneously — and additionally be capable of traversing paths that cross domain boundaries, which a domain-specific model cannot do because the path does not exist within its single-domain KG.

The medical model's superiority over GPT-5.2 is not evidence that medical specialization is required. It is evidence that GPT-5.2 was not trained with KG-structured process supervision. A general model with KG rewards across all domains would likely exceed the medical specialist on medical tasks (because it can draw on physics, chemistry, and mathematics when those are the mechanistic ground of the medical phenomenon) while also being capable on tasks the specialist cannot handle at all.

6b   Why the DSS Orchestrator Fails at Intelligence structural limitation

The DSS society architecture routes queries to domain experts via an orchestrator. The orchestrator decides which domain expert to invoke based on the apparent domain of the query. This architecture has a categorical failure mode that is not a tuning problem but a structural one.

Cross-domain insight requires recognizing, without prior knowledge, that a problem in Domain A has a solution structure from Domain B. The orchestrator can only route to Domain B if it already knows Domain B is relevant. But the insight — the recognition that Domain B applies — is precisely what cannot be computed without a unified representation space in which both domains are present and comparable.

Historical examples of this failure mode, rephrased as routing problems: Maxwell would not have unified electricity and magnetism if he was routing optics queries to an optics expert and electromagnetism queries to an EM expert. Watson and Crick would not have found DNA's structure if biology queries never reached the X-ray crystallography expert. The discovery of CRISPR required someone connecting bacteriophage defense (microbiology expert) to genetic engineering (molecular biology expert) — a connection the orchestrator would not make because neither query looks like it belongs in the other's domain.

Distant connections between fields are a key sign of genius precisely because they are not obvious to a routing system. They require a representation in which the structural similarity of superficially unrelated things is geometrically visible. That requires a unified representation space with cross-domain KG paths — not a collection of isolated spaces with a router in front.

6c   The Unified Multi-Domain KG plan

Instead of separate domain KGs fed into separate models, construct a single connected graph where domain KGs are components and cross-domain edges are explicit bridge relations.

Within-domain edges (dense)

  • Extracted by GraphMERT per domain
  • Well-typed, ontology-consistent
  • Verified factual with provenance
  • Multi-hop paths → training curriculum

Cross-domain bridge edges (sparser)

  • Shared mathematical structures (same governing equation)
  • Causal chains that cross domain boundaries
  • Structural analogies (isomorphic KG neighborhoods)
  • Concepts appearing in multi-domain literature

Cross-domain bridge edge discovery can be partially automated: find domain-concept pairs whose KG neighborhoods have high structural similarity (graph isomorphism tests on subgraphs); find concept co-occurrences across domain corpora; use the model itself (after initial training) to propose bridges by navigating toward structurally similar regions in its representation space. The quantum KG formalism (item 32) handles ambiguity: a cross-domain bridge edge can be a low-confidence superposition until confirmed by downstream training performance.

The unified KG then generates cross-domain path alignment training examples: problems whose solution requires traversal from a concept in one domain, through a bridge edge, into another domain, and back. These explicitly train analogical reasoning as a first-class skill.

6d   Residual-Coupled MoE Architecture directional

If domain-specific expert modules are still desirable (for specialization within a single model), the key constraint is that they must be able to communicate during processing — not just at input routing or output aggregation. Standard MoE experts are parallel and isolated. Residual Coupling (item 65) adds lateral residual connections between expert modules, solving this.

Each domain expert module receives: (1) the shared token representation from the encoder, (2) residual signals from other expert modules weighted by their KG proximity to the current concept. When the active concept in the unified KG has edges crossing into another domain, the coupling weight to that domain's expert is automatically upweighted. The experts are not isolated — they influence each other mid-computation in proportion to how KG-adjacent their domains are at this reasoning step.

This produces a model with the specialization benefits of MoE (each domain expert develops deep domain-specific representations) and the cross-domain benefits of a unified model (experts communicate when the KG topology says they should). The routing is no longer binary (invoke this expert, not that one) but continuous and KG-guided (all experts contribute with weights proportional to their KG proximity to the current concept).

Training signals for this architecture:

1
Standard pretraining objective — task completion or next-token prediction across diverse domains.
2
Within-domain path alignment reward — GRPO on KG paths within each domain (SFT + RL as in item 14). Expert modules specialize via this signal.
3
Cross-domain path alignment reward — GRPO on paths that traverse bridge edges in the unified KG. This is the signal that develops analogical reasoning and trains the Residual Coupling channels to transmit useful cross-domain information.
4
Coupling channel regularization — penalize channels that transmit noise (random high-entropy signals); encourage channels that carry KG-predictable information when adjacent domains are active. Prevents coupling from becoming spurious cross-talk.
Open design question
Coupling strength must be KG-topology-adaptive. Too strong → experts homogenize and lose domain structure. Too weak → effective isolation, same as standard MoE. See Q8.

6e   The Thousand Brains Parallel theoretical

Residual-Coupled domain experts within a unified model are structurally analogous to Thousand Brains Theory's cortical columns. In TBT: each column maintains its own model of the world using reference frames; columns vote and communicate laterally to reach a unified percept; no single column is authoritative. In the proposed architecture: each domain expert maintains its own structured representation of its domain using the KG geometry as a reference frame; experts communicate via residual coupling weighted by KG proximity; no single expert is the authoritative "answer generator."

The key TBT insight that applies here is the vote: the final output is a consensus across expert modules, not the output of whichever expert was selected by the router. This means cross-domain reasoning is not "asking the wrong expert" — it is all experts contributing a vote weighted by their KG proximity to the current concept, and the consensus naturally reflects whatever cross-domain structure exists.

In the latent-internalized version (Section 3c), the domain attractor regions are the column reference frames, and Residual Coupling channels are the lateral connections between columns. COCONUT latent reasoning traverses the attractor landscape guided by the unified KG topology — crossing from one domain's attractor region to another when the bridge edges point there.

Before implementing full Residual-Coupled MoE, first verify item 65's mechanism on a small experiment: two domain KGs, two expert modules, Residual Coupling between them, cross-domain path alignment reward. Test whether the coupling channels learn to carry KG-predictable information when cross-domain paths are active. This is a tractable small-scale test of the core architectural claim.
